### 题目 1：MDP基础建模 ⭐

**难度**: ⭐  
**涉及知识点**: MDP定义、转移模型、奖励函数

**题目**:  
考虑一个简单的网格世界：
```
+---+---+---+
| S |   | G |
+---+---+---+
|   | X |   |
+---+---+---+
```
- S: 起始状态(1,1)
- G: 目标状态(3,1)，奖励+10，终止
- X: 陷阱(2,2)，奖励-5，终止
- 其他格子: 每步奖励-1

动作: 上、下、左、右。每个动作以0.8概率成功，0.1概率向两侧滑动，0.1概率不动。撞墙则不动。

(a) 形式化定义这个MDP的五要素。

(b) 假设γ=0.9，写出状态(1,1)的贝尔曼方程(不需要求解)。

(c) 如果γ=0，最优策略是什么？

#### 逐步提示

<details>
<summary>点击查看提示</summary>

1. MDP = ⟨S, A, P, R, γ⟩
2. 注意区分终止状态和非终止状态
3. γ=0意味着只关心即时奖励
</details>

#### 详细解答

**(a) MDP五要素**

```
S = {(1,1), (1,2), (2,1), (2,2)=陷阱, (3,1)=目标, (3,2)}
    其中(2,2)和(3,1)是终止状态

A(s) = {Up, Down, Left, Right} 对所有非终止状态s

P(s'|s,a): 如P((1,2)|(1,1),Up) = 0.8
          P((2,1)|(1,1),Up) = 0.1 (向右滑动)
          P((1,1)|(1,1),Up) = 0.1 (撞左墙)

R(s,a,s'): -1 (非终止转移)
          +10 (到达G)
          -5 (到达X)
          
γ = 0.9 (给定)
```

**(b) 状态(1,1)的贝尔曼方程**

```
U(1,1) = max {
  Up:    0.8[-1 + 0.9U(1,2)] + 0.1[-1 + 0.9U(2,1)] + 0.1[-1 + 0.9U(1,1)],
  Down:  0.8[-1 + 0.9U(1,1)] + 0.1[-1 + 0.9U(2,1)] + 0.1[-1 + 0.9U(1,1)],
  Left:  0.9[-1 + 0.9U(1,1)] + 0.1[-1 + 0.9U(1,2)],
  Right: 0.8[-1 + 0.9U(2,1)] + 0.1[-1 + 0.9U(1,2)] + 0.1[-1 + 0.9U(1,1)]
}
```

**(c) γ=0时的最优策略**

```
γ=0意味着只考虑即时奖励

从(1,1):
- Up: 期望奖励 = 0.8×(-1) + 0.1×(-1) + 0.1×(-1) = -1
- Down: 同上 = -1
- Left: 0.9×(-1) + 0.1×(-1) = -1
- Right: 同上 = -1

所有动作期望相同！
但考虑陷阱风险:
- Up/Right可能滑向(2,1)，再一步可到G
- 但γ=0不考虑未来

最优策略: 任意(因为只关心即时奖励-1)
```

### 题目 2：贝尔曼方程求解 ⭐⭐

**难度**: ⭐⭐  
**涉及知识点**: 贝尔曼方程、价值迭代、最优策略

**题目**:  
考虑一个简化MDP，只有两个非终止状态A和B：
- 从A: 动作"Go"以概率1到达B，奖励+5
- 从B: 动作"Stay"以概率1留在B，奖励+1
- γ = 0.9

(a) 写出这个MDP的贝尔曼方程组。

(b) 求解状态A和B的最优价值。

(c) 提取最优策略。

#### 逐步提示

<details>
<summary>点击查看提示</summary>

1. U(B)的方程是递归的
2. 从B的最优方程求解U(B)
3. 然后代入求U(A)
</details>

#### 详细解答

**(a) 贝尔曼方程组**

```
U(A) = 5 + 0.9 × U(B)  // 只有Go一个动作

U(B) = max {
  Stay: 1 + 0.9 × U(B),  // 留在B
  Go:   ...               // 假设没有其他选择
}

假设只有Stay动作:
U(B) = 1 + 0.9 × U(B)
```

**(b) 求解**

```
从U(B) = 1 + 0.9U(B):
U(B) - 0.9U(B) = 1
0.1U(B) = 1
U(B) = 10

代入U(A):
U(A) = 5 + 0.9 × 10 = 5 + 9 = 14
```

**(c) 最优策略**

```
π*(A) = Go (唯一选择)
π*(B) = Stay (唯一选择)

解释:
- 在A，必须去B获得+5
- 在B，Stay每步获得+1，总价值10 = 1/(1-0.9)
  (几何级数求和)
```

### 题目 3：价值迭代过程 ⭐⭐

**难度**: ⭐⭐  
**涉及知识点**: 价值迭代、贝尔曼更新、收敛

**题目**:  
考虑以下3状态MDP(γ=0.5)：
- S₁: 动作a₁→S₂(奖励1)，动作a₂→S₃(奖励0)
- S₂: 动作→S₂(奖励2)  
- S₃: 动作→S₃(奖励0)  (S₂和S₃是自循环的终止状态)

执行价值迭代，初始U(S₁)=U(S₂)=U(S₃)=0。

(a) 计算前两次迭代后的价值。

(b) 当‖U_{i+1} - U_i‖_∞ < 0.01时停止，需要多少次迭代？

(c) 最优策略是什么？

#### 逐步提示

<details>
<summary>点击查看提示</summary>

1. 每次迭代对所有状态同时更新
2. S₂和S₃是自终止，价值可以直接计算
3. 使用无穷范数(最大绝对差)
</details>

#### 详细解答

**(a) 前两次迭代**

```
初始化: U₀ = [0, 0, 0]

S₂和S₃是直接终止状态:
U(S₂) = 2 + 0.5 × U(S₂) ⇒ U(S₂) = 4
U(S₃) = 0 + 0.5 × U(S₃) ⇒ U(S₃) = 0

实际上S₂和S₃从开始就确定:

第1次迭代:
U₁(S₁) = max {1 + 0.5×0, 0 + 0.5×0} = 1
U₁(S₂) = 4 (已收敛)
U₁(S₃) = 0 (已收敛)

第2次迭代:
U₂(S₁) = max {1 + 0.5×4, 0 + 0.5×0} = max{3, 0} = 3
```

**(b) 收敛分析**

```
继续迭代:
U₃(S₁) = max{1 + 0.5×4, 0} = 3  (已收敛!)

实际上S₁在第2次迭代后就稳定了。

但严格检查:
|U₂ - U₁| = |3 - 1| = 2 > 0.01
|U₃ - U₂| = 0 < 0.01

所以2次迭代后停止。

最优价值: U = [3, 4, 0]
```

**(c) 最优策略**

```
π*(S₁) = a₁ (去S₂，因为1 + 0.5×4 = 3 > 0)
π*(S₂) = 留在S₂ (奖励2)
π*(S₃) = 留在S₃ (奖励0)
```

### 题目 4：策略迭代 ⭐⭐⭐

**难度**: ⭐⭐⭐  
**涉及知识点**: 策略迭代、策略评估、策略改进

**题目**:  
考虑与题目3相同的MDP，使用策略迭代。

(a) 初始策略: π(S₁)=a₂, π(S₂)=Stay, π(S₃)=Stay。评估这个策略。

(b) 执行策略改进步骤。

(c) 策略迭代相比价值迭代有什么优势？

#### 逐步提示

<details>
<summary>点击查看提示</summary>

1. 策略评估解线性方程组(没有max)
2. 策略改进对每个状态选择最佳动作
3. 策略迭代通常收敛更快(迭代次数少)
</details>

#### 详细解答

**(a) 策略评估**

```
给定策略π:
π(S₁) = a₂ → S₃, 奖励0
π(S₂) = Stay, 奖励2
π(S₃) = Stay, 奖励0

解方程组:
U^π(S₁) = 0 + 0.5 × U^π(S₃)
U^π(S₂) = 2 + 0.5 × U^π(S₂) = 4
U^π(S₃) = 0 + 0.5 × U^π(S₃) = 0

因此:
U^π(S₁) = 0 + 0.5 × 0 = 0
U^π(S₂) = 4
U^π(S₃) = 0
```

**(b) 策略改进**

```
对每个状态s，选择:
a* = argmax_a Q(s, a)

S₁:
Q(S₁, a₁) = 1 + 0.5 × U^π(S₂) = 1 + 2 = 3
Q(S₁, a₂) = 0 + 0.5 × U^π(S₃) = 0

a₁更好！更新π(S₁) = a₁

S₂和S₃只有Stay动作。

新策略: π(S₁)=a₁, π(S₂)=Stay, π(S₃)=Stay

检查是否稳定:
用新策略重新评估，发现已是最优。
```

**(c) 策略迭代优势**

```
策略迭代优势:
1. 通常收敛更快(迭代次数少)
2. 每次迭代产生一个完整策略
3. 对于中间策略，智能体可以立即执行
4. 策略评估可以精确或近似

劣势:
1. 每次策略评估可能计算量大(O(|S|³)精确求解)
2. 对于大规模MDP，修正策略迭代更高效
```

### 题目 5：老虎机问题基础 ⭐⭐⭐

**难度**: ⭐⭐⭐  
**涉及知识点**: 老虎机问题、探索vs利用、基廷斯指数

**题目**:  
考虑一个2臂老虎机，折扣因子γ=0.5：
- 臂A: 确定产生序列: 4, 0, 0, 0, ...
- 臂B: 确定产生常数λ=1

(a) 如果只拉一个臂 forever，哪个臂更好？

(b) 最优策略是在某个时间点切换臂。找出最优切换时间和最大效用。

(c) 计算臂A的基廷斯指数。

#### 逐步提示

<details>
<summary>点击查看提示</summary>

1. 计算各臂的折现奖励总和
2. 尝试在T时刻从A切换到B
3. 基廷斯指数是最大平均折现奖励
</details>

#### 详细解答

**(a) 单一臂比较**

```
U(A) = 4 + 0.5×0 + 0.5²×0 + ... = 4
U(B) = 1 + 0.5×1 + 0.5²×1 + ... = 1/(1-0.5) = 2

单独看，A更好(4 > 2)
```

**(b) 最优切换策略**

```
如果在T=0切换到B:
U = 2 (纯B)

如果在T=1切换到B(拉A一次):
U = 4 + 0.5 × 2 = 4 + 1 = 5

如果在T=2切换到B(拉A两次):
U = 4 + 0.5×0 + 0.5² × 2 = 4 + 0.5 = 4.5

最优: T=1时切换，最大效用U = 5
```

**(c) 基廷斯指数**

```
基廷斯指数G(A)是使以下等式成立的λ:
max_T [Σ_{t=0}^{T-1} γ^t R_t] / [Σ_{t=0}^{T-1} γ^t] = λ/(1-γ)

或者等价地:
max_T [折现奖励] = λ/(1-γ) × [折现时间]

对于臂A:
T=1: 分子 = 4, 分母 = 1, 比值 = 4
T=2: 分子 = 4, 分母 = 1.5, 比值 = 2.67

最大比值在T=1: 4

G(A) × 1/(1-γ) = 4
G(A) = 4 × 0.5 = 2

解释: G(A) = 2意味着当λ=2时，A和B等价。
当λ < 2，选A；当λ > 2，选B。
```

### 题目 6：POMDP信念更新 ⭐⭐⭐⭐

**难度**: ⭐⭐⭐⭐  
**涉及知识点**: POMDP、信念状态、贝叶斯滤波

**题目**:  
考虑一个简化的机器人定位问题：
- 两个可能位置: L(左)和R(右)
- 动作: "TryLeft"尝试向左移动
- 转移: 成功概率0.8，失败(原地不动)概率0.2
- 传感器: 报告当前位置，准确率0.9

初始信念: b₀(L) = b₀(R) = 0.5

执行动作TryLeft，传感器报告"L"。

(a) 计算预测步骤后的信念(动作后，观测前)。

(b) 计算更新步骤后的信念(观测后)。

(c) 如果传感器报告"R"，计算最终信念。

#### 逐步提示

<details>
<summary>点击查看提示</summary>

1. 预测: b'(s') = Σ_s P(s'|s,a) b(s)
2. 更新: b''(s) = α P(e|s) b'(s)
3. 注意传感器可能出错
</details>

#### 详细解答

**(a) 预测步骤**

```
动作: TryLeft (尝试向左移动)

如果在L:
  P(到达L|L, TryLeft) = 0.2 (失败，原地)
  P(到达R|L, TryLeft) = 0 (不能向右)

如果在R:
  P(到达L|R, TryLeft) = 0.8 (成功左移)
  P(到达R|R, TryLeft) = 0.2 (失败)

预测信念:
b'(L) = P(L|L) b(L) + P(L|R) b(R)
      = 0.2 × 0.5 + 0.8 × 0.5
      = 0.1 + 0.4 = 0.5

b'(R) = P(R|L) b(L) + P(R|R) b(R)
      = 0 × 0.5 + 0.2 × 0.5
      = 0.1

等等，这不对，概率和应该为1。

正确计算:
b'(L) = 0.5
b'(R) = 0.5 - 等等，检查转移...

实际上，从R: 0.8概率到L，0.2概率留在R
所以:
b'(L) = 0.2×0.5 + 0.8×0.5 = 0.5
b'(R) = 0×0.5 + 0.2×0.5 = 0.1

但这和不为1。问题在于从L不能到R。

让我重新定义: 墙在左边界，TryLeft在L会失败(撞墙)

修正:
如果在L: TryLeft → 留在L (概率1，撞墙)
如果在R: TryLeft → L (0.8), R (0.2)

b'(L) = 1×0.5 + 0.8×0.5 = 0.9
b'(R) = 0×0.5 + 0.2×0.5 = 0.1
```

**(b) 观测更新(报告L)**

```
传感器模型:
P(报告L | 实际L) = 0.9
P(报告L | 实际R) = 0.1

b''(L) = α × 0.9 × 0.9 = α × 0.81
b''(R) = α × 0.1 × 0.1 = α × 0.01

归一化:
α × (0.81 + 0.01) = 1
α = 1/0.82 ≈ 1.22

b''(L) ≈ 0.988
b''(R) ≈ 0.012
```

**(c) 如果报告R**

```
b''(L) = α × P(报告R|L) × b'(L) = α × 0.1 × 0.9 = α × 0.09
b''(R) = α × P(报告R|R) × b'(R) = α × 0.9 × 0.1 = α × 0.09

b''(L) = b''(R) = 0.5

解释: 矛盾观测使机器人不确定！
```

### 题目 7：函数设计定理应用 ⭐⭐⭐⭐

**难度**: ⭐⭐⭐⭐  
**涉及知识点**: 奖励塑造、势函数、最优策略不变性

**题目**:  
考虑一个网格世界，目标是到达G。原始奖励：
- 每步: -1
- 到达G: +10

设计势函数Φ(s)来加速学习，但保持最优策略不变。

(a) 设计一个基于到G曼哈顿距离的势函数。

(b) 证明变换后的奖励鼓励向G移动。

(c) 讨论势函数设计的注意事项。

#### 逐步提示

<details>
<summary>点击查看提示</summary>

1. Φ(s) = -k × dist(s, G)
2. 新奖励 = 原奖励 + γΦ(s') - Φ(s)
3. 势函数应在目标处最大
</details>

#### 详细解答

**(a) 势函数设计**

```
设G位于(x_G, y_G)，当前位置(x, y)

曼哈顿距离: d(s) = |x - x_G| + |y - y_G|

势函数: Φ(s) = -d(s)

在G: Φ(G) = 0 (最大)
远离G: Φ(s) < 0
```

**(b) 奖励变换分析**

```
新奖励: R'(s,a,s') = R(s,a,s') + γΦ(s') - Φ(s)

考虑向G靠近一步的动作:
- 原奖励: -1
- 距离减少1: d(s') = d(s) - 1
- Φ(s') = -(d(s)-1) = -d(s) + 1 = Φ(s) + 1

新奖励:
R' = -1 + γ(Φ(s)+1) - Φ(s)
   = -1 + γΦ(s) + γ - Φ(s)
   = -1 + γ + (γ-1)Φ(s)
   = -(1-γ) + (γ-1)Φ(s)

如果γ=0.9:
R' = -0.1 - 0.1 × Φ(s)

因为Φ(s) < 0 (不在G)，-0.1 × Φ(s) > 0
所以R' > -0.1 > -1

向G移动的奖励比原来更大(惩罚更小)！
```

**(c) 设计注意事项**

```
1. 势函数必须在目标处取最大值
2. 势函数差异不能太大，否则可能数值不稳定
3. 势函数设计应反映领域知识
4. 多目标情况下设计更复杂
5. 势函数不改变最优策略，但影响学习速度
```

### 题目 8：老虎机与MDP的关系 ⭐⭐⭐⭐

**难度**: ⭐⭐⭐⭐  
**涉及知识点**: 老虎机超过程、分解、全局最优

**题目**:  
考虑一个老虎机超过程(BSP)有两个臂：
- 臂A: 3状态MDP，最优奖励序列: 5, 0, 0, 0, ...
- 臂B: 3状态MDP，最优奖励序列: 0, 0, 5, 0, ...

折扣因子γ=0.5。假设两个臂独立，且智能体一次只能使用一个臂。

(a) 分别求解每个臂的最优策略和价值。

(b) 证明全局最优策略可能与局部最优策略不同。

(c) 讨论机会成本如何影响决策。

#### 详细解答

**(a) 各臂最优策略**

```
臂A:
最优: 立即获得5，然后停止
U_A = 5

臂B:
最优: 等待3步获得5
U_B = 0.5² × 5 = 1.25

单独看，A远优于B
```

**(b) 全局最优 vs 局部最优**

```
如果必须只选一个臂:
选择A，总价值5

但考虑交替使用:
时间1: 使用A，获得5
时间2: 切换到B，开始积累
时间3: 使用B
时间4: 使用B，获得5

总价值: 5 + 0.5³ × 5 = 5 + 0.625 = 5.625 > 5!

全局最优策略涉及在两个臂间切换，
而不是始终使用单个臂的最优策略。

这是因为γ < 1: 等待有成本，
但通过交替可以并行化两个臂的"等待期"。
```

**(c) 机会成本分析**

```
机会成本 = 如果不使用当前臂，可以获得的价值

高机会成本 → 偏好即时奖励
低机会成本 → 可以等待延迟奖励

在BSP中:
- 臂A的机会成本高(延迟使用A意味着失去5)
- 臂B的机会成本低(延迟使用B只失去0.625)

全局最优考虑:
"我现在使用这个臂，放弃了其他臂的什么价值？"

这导致更"贪心"的行为，因为等待有折扣成本。
```

### 题目 9：实时动态规划与蒙特卡罗树搜索 ⭐⭐⭐⭐⭐

**难度**: ⭐⭐⭐⭐⭐  
**涉及知识点**: RTDP、UCT、在线规划、近似算法

**题目**:  
设计一个结合RTDP和UCT的混合算法用于大规模MDP。

**场景**: 机器人导航，状态空间10⁶，动作空间10。

(a) 解释为什么纯价值迭代不可行。

(b) 设计RTDP的启发式函数。

(c) 解释如何将UCT集成到框架中。

(d) 讨论计算资源分配策略。

#### 详细解答

**(a) 价值迭代不可行性**

```
|S| = 10⁶, |A| = 10

每次价值迭代: O(|S|²|A|) = O(10¹³)
不可行!

存储: |S|个价值 = 10⁶个浮点数 ≈ 8MB
尚可，但计算时间无法接受。
```

**(b) RTDP启发式设计**

```
启发式h(s) = 到目标的估计距离

选项:
1. 欧几里得距离 / 最大速度
2. 基于采样的路径估计
3. 学习得到的值函数近似

h(s)应该:
- 可容许: h(s) ≤ V*(s)
- 一致: h(s) ≤ R(s,a,s') + γh(s')

对于机器人导航:
h(s) = ||position(s) - goal|| / v_max
```

**(c) UCT集成**

```
┌────────────────────────────────────────────────────────┐
│                 混合算法框架                            │
├────────────────────────────────────────────────────────┤
│  在每个决策点:                                           │
│    1. 从当前状态s构建局部搜索树                          │
│    2. 使用UCT探索:                                       │
│       - 选择: UCB1公式                                   │
│       - 扩展: 添加新节点                                  │
│       - 模拟: 使用启发式策略到终止                        │
│       - 反向传播: 更新节点统计                            │
│    3. 执行最优动作                                        │
│    4. 观察结果，更新状态                                  │
│                                                         │
│  与RTDP结合:                                              │
│    - 使用RTDP的价值函数作为UCT的叶节点启发式              │
│    - UCT的探索结果用于更新RTDP的价值估计                  │
└────────────────────────────────────────────────────────┘
```

**(d) 资源分配策略**

```
计算预算B(如1000次模拟)

分配策略:
1. 固定深度: 限制树深度为d，在叶节点使用启发式
2. 自适应: 根据不确定性分配更多模拟
3.  anytime算法: 随时返回当前最佳动作

对于实时系统(如100Hz控制):
- 每步预算: 10ms
- 使用轻量级启发式
- 预计算常见模式
```

### 题目 10：POMDP与深度学习结合 ⭐⭐⭐⭐⭐

**难度**: ⭐⭐⭐⭐⭐  
**涉及知识点**: POMDP、深度学习、表示学习、端到端学习

**题目**:  
设计一个使用深度学习的POMDP求解框架。

**场景**: 自动驾驶中的多目标跟踪和决策。

(a) 如何用神经网络表示信念状态？

(b) 设计值网络(Q网络)的架构。

(c) 讨论训练策略和仿真环境的作用。

(d) 分析安全性和可解释性挑战。

#### 详细解答

**(a) 神经信念表示**

```
传统: 信念b(s) = P(状态=s | 历史)
      需要|S|维向量，对于大状态空间不可行

神经信念:
- 使用编码器网络压缩历史
- b = f_θ(观测序列)

架构选项:
1. RNN/LSTM: b_t = LSTM(o_t, b_{t-1})
2. Transformer: 注意力机制处理观测历史
3. 图神经网络: 处理结构化状态(多对象)

对于自动驾驶:
- 输入: 摄像头/雷达序列
- 编码: CNN提取特征 + RNN时序建模
- 输出: 潜在表示(低维信念)
```

**(b) Q网络架构**

```
┌────────────────────────────────────────────────────────┐
│                    Q-Network Architecture               │
├────────────────────────────────────────────────────────┤
│  Input:                                                │
│    - 当前观测图像/点云                                  │
│    - 历史观测(通过LSTM编码)                             │
│    - 当前动作候选                                       │
│                                                         │
│  Processing:                                           │
│    CNN特征提取 → 时序融合 → 动作嵌入 → 价值预测         │
│                                                         │
│  Output: Q(o, a) for all a ∈ A                         │
│                                                         │
│  或者(连续动作):                                       │
│    输入(o, a)，输出标量Q值                              │
│    需要优化argmax，可能使用策略网络辅助                   │
└────────────────────────────────────────────────────────┘
```

**(c) 训练策略**

```
训练数据:
1. 仿真环境生成:
   - 高保真物理仿真
   - 多样化场景
   - 危险场景专门采样

2. 真实世界数据(少量):
   - 人类驾驶数据
   - 离线强化学习

训练算法:
-  off-policy: DQN, DDPG, SAC
-  model-based: 学习世界模型，然后规划
-  imitation learning: 从专家演示学习

挑战:
- 分布偏移: 仿真到真实
- 安全探索: 不能在实际中尝试危险动作
- 奖励设计: 稀疏奖励，需要塑造
```

**(d) 安全与可解释性**

```
安全性挑战:
1. 神经网络的黑盒性质
   - 解决方案: 可验证的神经网络，区间传播

2. 分布外输入
   - 解决方案: 不确定性估计，异常检测

3. 对抗攻击
   - 解决方案: 对抗训练，输入验证

可解释性挑战:
1. 为什么采取这个动作？
   - 注意力可视化
   - 反事实解释

2. 信念表示什么？
   - 解码信念到语义空间
   - 生成语言描述

3. 验证和认证
   - 形式化方法验证关键属性
   - 人在回路验证
```

**1. 马尔可夫决策过程(MDP)与确定性搜索的主要区别是什么？**

A. MDP有目标状态，确定性搜索没有  
B. MDP的转移是随机的，确定性搜索的转移是确定的  
C. MDP使用启发式，确定性搜索不使用  
D. MDP只考虑即时奖励

<summary>答案</summary>

**答案: B**

**答案: B**

**3. 价值迭代算法收敛的原因是什么？**

A. 贝尔曼更新是压缩映射  
B. 奖励函数有界  
C. 状态空间有限  
D. 所有以上  

<details>
<summary>答案</summary>

**答案: D**

</details>

**4. 策略迭代相比价值迭代的主要优势是什么？**

A. 每次迭代计算量更小

C. 不需要折扣因子  
D. 可以处理连续状态  

<details>
<summary>答案</summary>

**答案: B**

策略迭代通常需要更少的迭代次数收敛，因为每次策略改进步骤可能直接到达或接近最优策略。

</details>

## 考点解析

### 核心概念对照表

| 考点 | 关键理解 | 常见错误 |
|-----|---------|---------|
| 贝尔曼方程 | 递归最优性 | 混淆V和Q |
| 价值迭代 | 压缩映射收敛 | 忽视γ<1条件 |
| 策略迭代 | 评估+改进交替 | 认为每次需精确评估 |
| POMDP信念 | 概率分布更新 | 只用最可能状态 |
| 基廷斯指数 | 可分解性 | 认为对所有问题适用 |
| UCB | 置信区间上界 | 与贪婪策略混淆 |
| 奖励塑造 | 势函数保持最优性 | 任意改变奖励 |

### 公式速查

```
贝尔曼最优方程:
V*(s) = max_a Σ P(s'|s,a)[R + γV*(s')]
Q*(s,a) = Σ P(s'|s,a)[R + γ max_a' Q*(s',a')]

价值迭代:
V_{k+1}(s) = max_a Σ P(s'|s,a)[R + γV_k(s')]

策略评估:
V^π(s) = Σ P(s'|s,π(s))[R + γV^π(s')]

基廷斯指数:
G(M,s) = (1-γ) × V*(M^s)

信念更新:
b'(s') = α P(e|s') Σ_s P(s'|s,a) b(s)
```
