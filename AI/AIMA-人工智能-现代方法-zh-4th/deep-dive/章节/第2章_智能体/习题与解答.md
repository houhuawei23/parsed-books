# 第2章 智能体 - 习题与解答

## 一、基础练习题

### 练习2.1 PEAS描述练习 ⭐
**难度**: 简单  
**涉及知识点**: PEAS模型、任务环境描述

**题目**: 为以下智能体创建PEAS描述：
(a) 足球守门员机器人  
(b) 在线书籍推荐系统  
(c) 家用自动扫地机器人

---

<details>
<summary>逐步提示</summary>

1. 对于每个智能体，分别考虑P、E、A、S四个要素
2. 性能度量应该量化"成功"的标准
3. 环境包括所有相关的外部因素
4. 执行器是智能体能做什么
5. 传感器是智能体能感知什么

</details>

<details>
<summary>详细解答</summary>

**(a) 足球守门员机器人**

| 要素 | 描述 |
|-----|------|
| **性能度量(P)** | 成功扑救次数、阻止进球数、反应时间、覆盖范围、与防守队员的配合 |
| **环境(E)** | 足球场、足球、对方球员、己方球员、天气条件、光照、球门区域、边界线 |
| **执行器(A)** | 机械腿（移动、跳跃、侧扑）、机械臂（阻挡）、身体转动 |
| **传感器(S)** | 摄像头（跟踪球和球员）、触觉传感器、加速度计、陀螺仪、距离传感器 |

**(b) 在线书籍推荐系统**

| 要素 | 描述 |
|-----|------|
| **性能度量(P)** | 用户点击率、购买转化率、用户满意度评分、推荐多样性、长期用户留存 |
| **环境(E)** | 用户、图书数据库、用户历史行为数据、流行趋势、其他用户行为、销售数据 |
| **执行器(A)** | 推荐列表展示、搜索排序、个性化页面生成、邮件推送 |
| **传感器(S)** | 用户浏览记录、点击流、购买历史、评分记录、搜索查询、停留时间 |

**(c) 家用自动扫地机器人**

| 要素 | 描述 |
|-----|------|
| **性能度量(P)** | 覆盖面积、清洁效率、电量消耗、噪音水平、避障成功率、清洁时间 |
| **环境(E)** | 房间布局、家具、地毯/硬地板、灰尘分布、宠物、门槛、楼梯、充电座位置 |
| **执行器(A)** | 驱动轮、吸尘器马达、边刷、充电对接机构 |
| **传感器(S)** | 碰撞传感器、悬崖传感器、摄像头/激光雷达、灰尘传感器、电量传感器、里程计 |

</details>

**考点解析**: PEAS是设计智能体的第一步，需要全面考虑任务环境的各个方面。常见错误包括：性能度量过于笼统（如只说"做得好"）、遗漏环境中的关键实体、传感器与执行器描述不完整。

### 练习2.2 理性判断 ⭐⭐
**难度**: 中等  
**涉及知识点**: 理性定义、理性四要素

**题目**: 考虑一个真空吸尘器智能体，它总是向右移动，从不吸尘。这个智能体在什么情况下是理性的？在什么情况下是不理性的？说明理由。

---

<details>
<summary>逐步提示</summary>

1. 回顾理性智能体的定义：最大化期望性能度量
2. 考虑不同的性能度量定义
3. 考虑不同的环境假设
4. 分析四要素：性能度量、先验知识、可选动作、感知序列

</details>

<details>
<summary>详细解答</summary>

**理性情况**：

1. **特殊性能度量**：如果性能度量是"尽可能向右移动的次数"，且方格永远保持干净，则该智能体是理性的。

2. **环境约束**：如果环境只有一个方格（无灰尘），且性能度量惩罚吸尘动作（如噪音成本），则向右移动然后停止可能是理性的。

3. **知识缺失**：如果智能体完全不知道吸尘动作的存在，在其知识范围内它已做出最优选择。

**不理性情况**：

1. **标准真空环境**：如果性能度量是"清理的灰尘量"或"干净方格数"，且方格会变脏，则该智能体是不理性的，因为它完全忽略了清洁任务。

2. **有完备知识**：如果智能体知道吸尘可以清洁方格但选择不吸尘，且清洁方格对性能度量有益，则是不理性的。

3. **对比分析**：任何能吸尘的智能体在该环境下都会表现更好，因此"总是向右"的策略不可能是理性的。

**结论**：判断理性必须明确四要素。同一个行为在不同假设下可能理性也可能不理性。

</details>

**考点解析**: 本题强调理性是相对于特定任务环境的。理性≠智能，理性≠完美。必须根据具体假设判断，不能脱离上下文。

---

### 练习2.3 环境分类 ⭐⭐
**难度**: 中等  
**涉及知识点**: 环境七维度分类

**题目**: 对以下任务环境进行分类（使用7个维度）：

(a) 井字棋（Tic-Tac-Toe）
(b) 医疗诊断系统  
(c) 自动驾驶汽车  
(d) 图像分类器（识别照片中的动物）

---

<details>
<summary>逐步提示</summary>

对每个环境，分析：
1. 可观测性：能看到什么？
2. 智能体数量：有多少决策者？
3. 确定性：结果是否确定？
4. 回合/序贯：决策是否相互依赖？
5. 动态性：思考时环境变化吗？
6. 离散/连续：状态和动作是离散还是连续？
7. 已知/未知：了解规则吗？

</details>

<details>
<summary>详细解答</summary>

**(a) 井字棋**

| 维度 | 分类 | 说明 |
|-----|------|------|
| 可观测性 | 完全可观测 | 双方都能看到整个棋盘 |
| 智能体数 | 多智能体（竞争） | 两名玩家，目标相反 |
| 确定性 | 确定性 | 落子结果完全确定 |
| 回合/序贯 | 序贯 | 当前落子影响未来局面 |
| 动态性 | 静态 | 对手等待你落子 |
| 离散/连续 | 离散 | 有限状态和动作 |
| 已知/未知 | 已知 | 规则完全明确 |

**(b) 医疗诊断系统**

| 维度 | 分类 | 说明 |
|-----|------|------|
| 可观测性 | 部分可观测 | 无法直接观察内部病变 |
| 智能体数 | 单/多智能体 | 主要单智能体，但涉及病人、医生等 |
| 确定性 | 非确定性 | 同病症可能有不同发展 |
| 回合/序贯 | 序贯 | 诊断和治疗是持续过程 |
| 动态性 | 动态 | 病情随时间变化 |
| 离散/连续 | 连续 | 许多生理指标是连续的 |
| 已知/未知 | 部分已知 | 医学知识有限 |

**(c) 自动驾驶汽车**

| 维度 | 分类 | 说明 |
|-----|------|------|
| 可观测性 | 部分可观测 | 看不到其他司机意图 |
| 智能体数 | 多智能体（部分合作） | 其他车辆、行人、交通信号 |
| 确定性 | 非确定性/随机 | 交通行为难以预测 |
| 回合/序贯 | 序贯 | 当前驾驶决策影响未来 |
| 动态性 | 动态 | 其他车辆持续移动 |
| 离散/连续 | 连续 | 位置、速度都是连续的 |
| 已知/未知 | 大部分已知 | 交通规则已知，但具体路况未知 |

**(d) 图像分类器**

| 维度 | 分类 | 说明 |
|-----|------|------|
| 可观测性 | 完全可观测 | 能获取完整图像 |
| 智能体数 | 单智能体 | 独立完成任务 |
| 确定性 | 确定性 | 给定图像输出确定 |
| 回合/序贯 | 回合式 | 每张图像独立分类 |
| 动态性 | 半动态 | 图像不变但处理时间影响性能 |
| 离散/连续 | 连续输入，离散输出 | 像素值连续，类别标签离散 |
| 已知/未知 | 已知 | 分类规则由训练确定 |

</details>

**考点解析**: 环境分类是选择适当智能体设计的前提。注意：(1) 已知≠可观测；(2) 回合式可以有多个感知-动作，关键是决策之间无依赖；(3) 多智能体判断关键是其他实体是否有目标。

---

### 练习2.4 理性vs全知 ⭐⭐
**难度**: 中等  
**涉及知识点**: 理性定义、全知概念、期望vs实际性能

**题目**: 解释以下说法是否正确："如果一个智能体是理性的，它就不应该犯错。" 请用书中香榭丽舍大街的例子说明。

---

<details>
<summary>逐步提示</summary>

1. 明确理性和全知的区别
2. 理解"期望性能"和"实际性能"的区别
3. 分析香榭丽舍大街例子
4. 讨论"犯错"的定义

</details>

<details>
<summary>详细解答</summary>

**该说法不正确**。

**关键区别**：
- **理性**：最大化期望性能（基于已有信息做出的最优决策）
- **全知**：最大化实际性能（知道未来的实际结果）
- **理性≠全知**：理性不要求预知能力

**香榭丽舍大街例子分析**：
- 场景：你看到对面有老朋友，附近无车，开始过马路
- 意外：10km高空货舱门掉落
- 结果：被压扁

**判断**：过马路是理性的，因为：
1. 基于当时的感知序列（朋友、无车），过马路是合理的
2. 智能体无法预知货舱门掉落这种极低概率事件
3. 如果不观察路况就过马路才是不理性的

**关于"犯错"**：
- 如果"犯错"指"事后看来不是最优的"，则理性智能体会"犯错"
- 如果"犯错"指"基于当时信息做出次优决策"，则理性智能体不会犯错

**结论**：理性是关于决策过程的属性，而非结果的属性。好的决策可能导致坏的结果（由于不确定性）。

</details>

**考点解析**: 这是哲学上的重要区分。考试中常考此概念，注意用"期望"vs"实际"、"事前"vs"事后"的对比来理解。

### 练习2.5 条件-动作规则设计 ⭐⭐
**难度**: 中等  
**涉及知识点**: 简单反射型智能体、条件-动作规则

**题目**: 设计一个简单反射型智能体的条件-动作规则，用于在3×3网格环境中收集垃圾。智能体能感知当前位置是否有垃圾，以及向四个方向移动时是否碰到墙。

---

<details>
<summary>逐步提示</summary>

1. 确定感知内容：当前位置是否有垃圾、四个方向是否可移动
2. 确定动作：捡垃圾、向四个方向移动
3. 设计优先级：捡垃圾优先，然后探索
4. 考虑边界情况：遇到墙时的处理

</details>

<details>
<summary>详细解答</summary>

**感知表示**：
- `Garbage`: 当前位置有垃圾（布尔值）
- `NorthClear`, `SouthClear`, `EastClear`, `WestClear`: 各方向是否可移动（布尔值）

**条件-动作规则集**（按优先级排序）：

```
规则1: IF    Garbage 
       THEN  PickUp

规则2: IF    NorthClear AND NOT SouthClear 
       THEN  MoveNorth

规则3: IF    EastClear AND NOT WestClear  
       THEN  MoveEast

规则4: IF    SouthClear AND NOT NorthClear
       THEN  MoveSouth

规则5: IF    WestClear AND NOT EastClear
       THEN  MoveWest

规则6: IF    NorthClear
       THEN  MoveNorth

规则7: IF    EastClear
       THEN  MoveEast

规则8: IF    SouthClear
       THEN  MoveSouth

规则9: IF    WestClear
       THEN  MoveWest
```

**规则说明**：
- 规则1：有垃圾先捡垃圾
- 规则2-5：优先向未探索过的方向移动（假设之前从相反方向来）
- 规则6-9：默认移动方向优先级

**局限性**：
这是一个简单的反射型智能体，在部分可观测环境中可能陷入循环。更好的设计需要内部状态来记录已访问的位置。

</details>

**考点解析**: 简单反射型智能体适合完全可观测环境。设计时应注意规则的完备性和优先级。在部分可观测环境中，简单反射型智能体容易陷入无限循环。

## 二、综合练习题

### 练习2.6 多维度分析 ⭐⭐⭐
**难度**: 较难  
**涉及知识点**: PEAS、环境分类、智能体类型选择

**题目**: 考虑一个**扑克牌游戏**智能体：

(a) 给出完整的PEAS描述  
(b) 分析其环境的所有维度  
(c) 说明为什么简单反射型智能体不适合该任务  
(d) 推荐一种智能体类型并说明理由

---

<details>
<summary>逐步提示</summary>

(a) 考虑扑克游戏的特殊元素：运气、对手、隐藏信息  
(b) 逐条分析7个维度，注意"部分可观测"和"多智能体竞争"  
(c) 从简单反射型的局限性分析  
(d) 考虑需要什么样的推理能力

</details>

<details>
<summary>详细解答</summary>

**(a) PEAS描述**

| 要素 | 描述 |
|-----|------|
| **性能度量(P)** | 赢得的筹码数、长期胜率、对抗特定对手的表现、风险控制（避免破产） |
| **环境(E)** | 牌桌、其他玩家、庄家/发牌员、牌堆、下注规则、游戏历史、时间限制 |
| **执行器(A)** | 下注、跟注、加注、弃牌、过牌、显示手牌 |
| **传感器(S)** | 可见的公共牌、自己的手牌、其他玩家的可见行为（下注模式、时间、表情）、筹码数量 |

**(b) 环境维度分析**

| 维度 | 分类 | 具体说明 |
|-----|------|---------|
| 可观测性 | **部分可观测** | 看不到对手的手牌和牌堆中的牌 |
| 智能体数 | **多智能体（竞争）** | 多个玩家，目标都是赢得对方的筹码 |
| 确定性 | **随机性** | 发牌是随机的，但玩家决策基于概率 |
| 回合/序贯 | **序贯** | 当前下注影响后续轮次和对手策略 |
| 动态性 | **静态** | 思考时环境不变（除非是限时扑克） |
| 离散/连续 | **离散** | 有限的牌组合和离散的下注动作 |
| 已知/未知 | **已知** | 规则已知，但对手策略未知 |

**(c) 简单反射型不适合的原因**

1. **部分可观测性**：简单反射型只基于当前感知决策，无法利用历史信息推断对手手牌
2. **序贯决策**：当前决策影响未来，需要前瞻规划
3. **多智能体**：需要考虑对手可能的行为和策略
4. **概率推理**：需要根据不完全信息计算期望值
5. **对手建模**：需要维护对手行为模式的内部状态

**(d) 推荐智能体类型**

推荐：**基于效用的智能体（结合学习元素）**

理由：
1. **基于模型**：需要维护对手模型、牌堆概率模型
2. **基于效用**：需要权衡风险与收益，不同牌局需要不同的风险态度
3. **学习元素**：需要学习对手策略模式、调整自己的策略
4. **期望效用最大化**：在不确定性下做出最优决策

这种智能体能够：
- 估计手牌胜率
- 推断对手可能的手牌范围
- 根据对手类型调整策略
- 从经验中改进决策

</details>

**考点解析**: 本题综合考察PEAS、环境分类和智能体选择。关键理解：环境特性直接决定智能体设计需求。部分可观测+序贯+多智能体+随机 = 需要复杂智能体。

---

### 练习2.7 智能体结构分析 ⭐⭐⭐
**难度**: 较难  
**涉及知识点**: 四种智能体类型、内部组件、适用场景

**题目**: 比较以下四种智能体在**仓库搬运机器人**任务中的优缺点：
(a) 简单反射型  
(b) 基于模型的反射型  
(c) 基于目标的  
(d) 基于效用的

任务描述：机器人需要在仓库中搬运货物，避免碰撞，优化路径，可能有多辆机器人协作。

---

<details>
<summary>逐步提示</summary>

1. 分析任务需求：避障、路径规划、多机器人协调
2. 每种智能体分析：优点、缺点、适用性
3. 比较：信息需求、计算复杂度、灵活性

</details>

<details>
<summary>详细解答</summary>

**(a) 简单反射型智能体**

*优点*：
- 响应快速，实时性好
- 实现简单，计算开销小
- 适合紧急情况反应

*缺点*：
- 无法处理部分可观测性（遮挡的障碍物）
- 无法进行路径规划（需要目标）
- 无法协调多机器人行动
- 容易陷入局部最优或循环

*适用性*：**不适合**作为主要控制，仅可用于紧急停止等简单反射

---

**(b) 基于模型的反射型智能体**

*优点*：
- 能处理部分可观测性（维护位置估计）
- 能预测环境变化（其他机器人移动）
- 比简单反射型更智能

*缺点*：
- 没有明确目标，可能无目的游荡
- 无法进行长期路径规划
- 难以优化整体效率

*适用性*：**部分适合**，可作为基础控制层，但需要上层规划

---

**(c) 基于目标的智能体**

*优点*：
- 能进行路径规划（从当前位置到目标位置）
- 能协调多个任务（取货→送货）
- 行为可预测、可解释

*缺点*：
- 无法处理目标冲突（如时间vs能量）
- 无法评估不同路径的质量差异
- 在不确定性下难以决策

*适用性*：**适合**确定性较强的环境，适合单一目标优化

---

**(d) 基于效用的智能体**

*优点*：
- 能权衡多目标（时间、能耗、安全）
- 能在不确定性下做出最优决策
- 能评估不同方案的优劣
- 适合多机器人协调（分配任务最大化整体效用）

*缺点*：
- 计算复杂度高
- 需要准确的效用函数设计
- 实时性挑战

*适用性*：**最适合**复杂仓库环境，尤其是动态、多机器人场景

---

**综合建议**：
实际系统可采用分层架构：
- 底层：基于模型的反射型（避障、安全）
- 中层：基于目标的（路径规划）
- 顶层：基于效用的（任务调度、资源分配）

</details>

**考点解析**: 智能体类型选择需要考虑任务复杂度、环境特性和性能要求。没有绝对最好的类型，只有最适合的。实际系统常常是多种类型的组合。

---

### 练习2.8 环境特性影响分析 ⭐⭐⭐
**难度**: 较难  
**涉及知识点**: 环境维度、智能体设计影响

**题目**: 解释以下环境特性如何影响智能体设计：

(a) 从**完全可观测**变为**部分可观测**  
(b) 从**静态**变为**动态**  
(c) 从**单智能体**变为**多智能体（竞争）**  
(d) 从**确定性**变为**随机性**

对每种情况，说明：智能体需要增加什么能力？设计复杂度如何变化？

---

<details>
<summary>逐步提示</summary>

(a) 思考：看不到全部时需要什么？  
(b) 思考：环境变化时决策有什么要求？  
(c) 思考：有对手时需要考虑什么？  
(d) 思考：结果不确定时如何决策？

</details>

<details>
<summary>详细解答</summary>

**(a) 完全可观测 → 部分可观测**

*需要增加的能力*：
- **状态估计**：维护内部状态来跟踪不可观测变量
- **记忆**：存储历史感知来推断当前状态
- **传感器模型**：理解感知与真实状态的关系

*设计复杂度*：
- **显著增加**：从简单查找表变为需要状态更新逻辑
- 需要处理不确定性（概率推理）
- 可能永远不确定真实状态，只能维护信念分布

*例子*：自动驾驶汽车无法看到其他司机想法，需要预测其行为

---

**(b) 静态 → 动态**

*需要增加的能力*：
- **实时决策**：必须在有限时间内做出决策
- **持续监控**：不能"离线"思考太久
- **反应速度**：对环境变化快速响应

*设计复杂度*：
- **中等增加**：需要 anytime algorithm（随时算法）
- 可能需要将计算与动作交织
- 决策质量与时间的权衡

*例子*：静态的填字游戏可以深思熟虑，动态的驾驶需要即时反应

---

**(c) 单智能体 → 多智能体（竞争）**

*需要增加的能力*：
- **对手建模**：预测其他智能体的行为
- **博弈推理**：考虑对手也在推理我
- **随机化**：避免被对手预测
- **通信**：可能需要协商或欺骗

*设计复杂度*：
- **大幅增加**：从优化问题变为博弈问题
- 可能需要纳什均衡等博弈论概念
- 对手行为的不确定性增加了复杂性

*例子*：国际象棋vs解谜游戏，需要考虑对手策略

---

**(d) 确定性 → 随机性**

*需要增加的能力*：
- **概率推理**：处理结果分布而非确定结果
- **期望计算**：计算动作的期望效用
- **风险建模**：理解不同结果的概率
- **信息价值**：评估减少不确定性的动作

*设计复杂度*：
- **显著增加**：决策基于概率分布而非确定值
- 需要准确的概率模型
- 可能需要贝叶斯更新等复杂计算

*例子*：确定性迷宫vs涉及骰子或天气的决策

---

**综合影响**：

最困难的组合是：**部分可观测 + 动态 + 多智能体 + 随机性**

这正是真实世界的自动驾驶面临的挑战！

</details>

**考点解析**: 环境维度是智能体设计的决定因素。理解每个维度变化带来的设计需求变化，是选择合适AI技术的关键。

要求：
1. 识别系统中涉及的所有智能体类型（至少3种）

1. 识别系统中的参与者：交通信号灯、自动驾驶车、公交调度、应急车辆等
2. 每种参与者都是一个智能体

**智能体2：自动驾驶车辆**

| **执行器** | 转向、加速、刹车、灯光、喇叭、通信设备 |
| **传感器** | 摄像头、激光雷达、雷达、GPS、IMU、V2X通信 |

**智能体3：公交调度系统**

| 要素 | 描述 |

**智能体4：应急车辆优先系统**

| 要素 | 描述 |
|-----|------|

| **环境** | 道路网络、交通状况、医院位置、事故现场、其他车辆 |
| **执行器** | 路线规划、信号优先请求、警报系统、车道控制 |
| **传感器** | GPS、交通数据、事故报告、医院状态 |

**智能体交互关系**

```

- 合作：共享数据优化整体流量
- 竞争：道路资源分配
- 层次：应急优先 > 公交 > 普通车辆
```

**性能度量冲突与解决方案**

**系统设计挑战**：

2. **实时协调**：大规模分布式决策
3. **不确定性**：需求预测、事故处理
4. **公平性**：不同用户群体的利益平衡
5. **隐私**：位置数据的收集与使用

</details>

**考点解析**: 复杂系统中的PEAS分析需要考虑多个智能体及其交互。性能度量的设计尤为关键，需要平衡多方利益。这是AI在真实世界应用的核心挑战。

### 练习2.10 知识表示演进分析 ⭐⭐⭐⭐
**难度**: 困难  
**涉及知识点**: 原子、因子化、结构化表示、表达能力vs推理复杂度

**题目**: 考虑**餐厅推荐系统**的知识表示，说明以下三种表示方式的区别和演进：

(a) 原子表示：将每个"餐厅-用户-情境"组合视为一个原子状态  
(b) 因子化表示：使用属性向量描述餐厅和用户  
(c) 结构化表示：使用对象和关系建模

对每种表示：
1. 给出具体表示示例
2. 分析表达能力（能表示什么知识）
3. 分析推理复杂度
4. 讨论学习可行性

---

<details>
<summary>逐步提示</summary>

1. 原子：没有内部结构，每个组合是独立的
2. 因子化：分解成属性值
3. 结构化：对象、属性、关系
4. 考虑推荐系统需要的知识类型

</details>

<details>
<summary>详细解答</summary>

**(a) 原子表示**

*表示示例*：
```
状态空间 = 所有可能的(餐厅, 用户, 时间, 位置)组合

例如：
- 状态1：(海底捞, 用户A, 周五晚上, 三里屯) → 推荐
- 状态2：(海底捞, 用户A, 周一中午, 三里屯) → 不推荐
- 状态3：(必胜客, 用户A, 周五晚上, 三里屯) → 不推荐
...
```

*表达能力*：
- 只能表示"在什么情况下是否推荐"
- 无法表示"为什么推荐"
- 每个组合独立，无法泛化

*推理复杂度*：
- 查找：O(1)
- 但需要存储的组合数是指数级的
- 无法处理未见过的组合

*学习可行性*：
- 需要大量的训练数据覆盖所有组合
- 无法从已见组合推断未见组合
-  practically infeasible

---

**(b) 因子化表示**

*表示示例*：
```
餐厅 = {菜系: 川菜, 价格: 高, 评分: 4.5, 距离: 2km, ...}
用户 = {偏好: [辣, 火锅], 预算: 200, 当前位置: (x,y), ...}
情境 = {时间: 19:00, 日期: 周五, 天气: 晴, ...}

特征向量 = [菜系匹配, 价格差, 评分, 距离, 时间偏好匹配, ...]
```

*表达能力*：
- 能表示属性间的相似性
- 可以泛化到相似的餐厅
- 能学习属性权重（如价格重要性）

*推理复杂度*：
- 特征提取：O(属性数)
- 相似度计算：O(特征维度)
- 需要特征工程

*学习可行性*：
- 可以学习特征权重
- 能泛化到新餐厅（如果属性已知）
- 常用方法：协同过滤、矩阵分解

---

**(c) 结构化表示**

*表示示例*：
```
对象：
- 餐厅R1: 类型=火锅, 位置=L1, 菜系=川菜
- 用户U1: 喜欢=[R2, R3], 朋友=U2
- 位置L1: 区域=三里屯, 地铁=团结湖站
- 菜品D1: 类型=毛肚, 辣度=高, 属于=R1

关系：
- 距离(L1, L2) = 3km
- 相似(R1, R2) = 0.8  (因为都是火锅)
- U1喜欢D1 (因为U1喜欢辣且D1是辣的)
- U2去了R1 (朋友影响)

规则：
- IF 朋友喜欢X AND 相似(用户, 朋友) THEN 可能喜欢X
- IF 喜欢辣 AND 菜品辣 THEN 喜欢菜品
```

*表达能力*：
- 能表示对象间复杂关系
- 能进行关系推理（朋友的朋友）
- 能表示组合关系（菜品-餐厅-菜系）

*推理复杂度*：
- 图遍历：O(节点+边)
- 逻辑推理：可能指数级
- 需要复杂的推理算法

*学习可行性*：
- 知识图谱嵌入
- 关系学习
- 图神经网络
- 能发现隐含关系

---

**对比总结**

| 方面 | 原子 | 因子化 | 结构化 |
|-----|------|--------|--------|
| 表达简洁性 | 差（需要枚举） | 中 | 好（关系共享） |
| 泛化能力 | 无 | 中 | 强 |
| 推理效率 | 查找快 | 中等 | 可能慢 |
| 学习难度 | 数据饥渴 | 中等 | 需要大量技术 |
| 可解释性 | 无 | 中 | 好（关系明确） |

**实际应用**：
现代推荐系统通常结合使用：
- 因子化：用户/物品嵌入
- 结构化：知识图谱增强
- 深度学习：自动特征学习

</details>

**考点解析**: 三种表示的权衡是AI的核心主题。表达能力越强，推理和学习通常越复杂。实际系统需要根据任务选择合适的表示或组合使用。

## 答题要点速查

### PEAS记忆口诀
```
P - 做得怎么样？（Performance）
E - 在哪里做？（Environment）  
A - 用什么做？（Actuators）
S - 怎么知道？（Sensors）
```

### 环境七维度速记
```
观测（完全/部分）× 数量（单/多）× 确定（确定/随机）
× 结构（回合/序贯）× 动态（静/动）× 连续（离散/连续）
× 知识（已知/未知）
```

### 智能体类型选择指南
```
完全可观测 + 简单任务 → 简单反射型
部分可观测 → 基于模型的反射型
需要目标导向 → 基于目标
多目标/不确定性 → 基于效用
需要改进 → 学习型
```
