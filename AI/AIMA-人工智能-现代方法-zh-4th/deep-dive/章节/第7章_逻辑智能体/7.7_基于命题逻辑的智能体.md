# 7.7 基于命题逻辑的智能体 (Propositional Logic Agent)

## 1. 背景与动机

### 1.1 历史背景

基于逻辑的智能体是人工智能早期研究的核心方向。从20世纪60年代的通用问题求解器（GPS）到70年代的专家系统，逻辑推理一直是实现智能行为的主要方法。

**早期发展**：
- **1960年代**：Newell和Simon的GPS使用逻辑推理来解决问题。
- **1970-1980年代**：专家系统（如MYCIN、DENDRAL）使用基于规则的推理在特定领域达到专家水平。
- **1990年代**：规划系统（如STRIPS、PDDL）使用逻辑表示世界状态和动作。

**现代发展**：
- **SATPlan**（Kautz and Selman, 1992, 1996）：将规划问题编码为SAT问题，利用现代SAT求解器高效求解。
- **混合智能体**：结合逻辑推理和问题求解搜索，充分利用两者的优势。
- **知识图谱**：大规模结构化知识库，支持复杂推理。

### 1.2 研究动机

构建基于命题逻辑的智能体有以下动机：

**（1）形式化保证**：逻辑推理保证结论的正确性，只要知识库为真，推导出的结论也必然为真。

**（2）可解释性**：智能体的决策过程可以被解释和验证，每个结论都有明确的推理链。

**（3）知识重用**：相同的推理引擎可以应用于不同领域，只需更换知识库。

**（4）自动规划**：通过逻辑推理可以自动生成达成目标的规划。

### 1.3 应用场景

基于命题逻辑的智能体在以下场景有应用：

| 应用领域 | 具体应用 | 技术特点 |

## 3. 核心概念与数学分析

### 3.1 术语定义

| 术语（中文） | 术语（英文） | 定义 |
|------------|-------------|------|
| 流 | Fluent | 随时间变化的世界属性，用时间上标标记 |
| 非时序变量 | Atemporal Variable | 不随时间变化的世界属性 |
| 效应公理 | Effect Axiom | 描述动作效果的公理 |
| 框架公理 | Frame Axiom | 描述动作未改变状态的公理 |
| 后继状态公理 | Successor-State Axiom | 综合效应和框架的公理 |
| 前提公理 | Precondition Axiom | 描述动作执行前提的公理 |
| 动作排除公理 | Action Exclusion Axiom | 禁止同时执行冲突动作的公理 |

### 3.3 时序知识表示

#### 3.3.1 感知的时间标记

为了避免新旧感知之间的矛盾，每个感知都用时间步标记：

- $Stench^4$：时刻4感知到臭味
- $¬Breeze^3$：时刻3没有感知到微风

这样可以避免与$¬Stench^3$产生矛盾。

#### 3.3.2 流（Fluents）

流是随时间变化的世界属性，用时间上标标记：

- $L_{1,1}^0$：时刻0智能体位于$[1,1]$
- $FacingEast^0$：时刻0智能体面朝东
- $HaveArrow^t$：时刻$t$智能体有箭

#### 3.3.3 非时序变量

与世界不变部分相关的符号不需要时间上标：

- $P_{x,y}$：$[x,y]$有无底洞（恒定）
- $W_{x,y}$：wumpus在$[x,y]$（恒定，除非被杀死）
- $B_{x,y}$：$[x,y]$有微风（恒定）

### 3.4 动作表示

#### 3.4.1 效应公理

效应公理描述动作产生的结果：

**前进动作效应**：
$$L_{1,1}^0 \land FacingEast^0 \land Forward^0 \Rightarrow (L_{2,1}^1 \land ¬L_{1,1}^1)$$

含义：如果智能体在时刻0位于$[1,1]$且面朝东并执行前进动作，则在时刻1位于$[2,1]$且不在$[1,1]$。

#### 3.4.2 框架问题

框架问题是如何表示动作未改变的状态。

**表示框架问题**：需要$O(mn)$个框架公理，其中$m$是动作数，$n$是流数。

**框架公理示例**：
$$Forward^t \Rightarrow (HaveArrow^t \Leftrightarrow HaveArrow^{t+1})$$

含义：前进动作不改变是否有箭的状态。

#### 3.4.3 后继状态公理

后继状态公理解决了框架问题，将公理集大小从$O(mn)$减少到$O(mk)$，其中$k$是每个动作改变的平均流数。

**一般形式**：
$$F^{t+1} \Leftrightarrow ActionCausesF^t \lor (F^t \land ¬ActionCausesNotF^t)$$

含义：流$F$在$t+1$时刻为真，当且仅当：
- 时刻$t$的动作导致$F$为真，或
- $F$在时刻$t$已经为真且动作没有导致它为假

**HaveArrow的后继状态公理**：
$$HaveArrow^{t+1} \Leftrightarrow (HaveArrow^t \land ¬Shoot^t)$$

含义：有箭当且仅当之前有箭且没有射击。

**位置的后继状态公理**（更复杂）：
$$L_{1,1}^{t+1} \Leftrightarrow (L_{1,1}^t \land (¬Forward^t \lor Bump^{t+1})) \lor (L_{1,2}^t \land FacingSouth^t \land Forward^t) \lor (L_{2,1}^t \land FacingWest^t \land Forward^t)$$

含义：在$[1,1]$当且仅当：
- 之前在$[1,1]$且没有向前移动（或撞墙），或
- 从$[1,2]$向南移动，或
- 从$[2,1]$向西移动

### 3.5 混合智能体

#### 3.5.1 架构

混合智能体结合逻辑推断和问题求解搜索：

```
function HYBRID-WUMPUS-AGENT(percept) returns an action
    persistent: 
        KB, 知识库，初始为非时序的"wumpus物理学"
        t, 计数器，初始为0
        plan, 动作序列，初始为空
    
    TELL(KB, MAKE-PERCEPT-SENTENCE(percept, t))
    告知KB对于时刻t的时序"physics"语句
    
    safe ← {[x,y]: ASK(KB, OK_{x,y}^t) = true}
    
    if ASK(KB, Glitter^t) = true then
        plan ← [Grab] + PLAN-ROUTE(current, [1,1], safe) + [Climb]
    
    if plan为空 then
        unvisited ← {[x,y]: ASK(KB, L_{x,y}^{t'}) = false 对所有t' ≤ t}
        plan ← PLAN-ROUTE(current, unvisited ∩ safe, safe)
    
    if plan为空且ASK(KB, HaveArrow^t) = true then
        possible_wumpus ← {[x,y]: ASK(KB, ¬W_{x,y}) = false}
        plan ← PLAN-SHOT(current, possible_wumpus, safe)
    
    if plan为空 then
        not_unsafe ← {[x,y]: ASK(KB, ¬OK_{x,y}^t) = false}
        plan ← PLAN-ROUTE(current, unvisited ∩ not_unsafe, safe)
    
    if plan为空 then
        plan ← PLAN-ROUTE(current, [1,1], safe) + [Climb]
    
    action ← POP(plan)
    TELL(KB, MAKE-ACTION-SENTENCE(action, t))
    t ← t + 1
    return action
```

#### 3.5.2 决策层次

智能体按以下优先级创建规划：

1. **抓取金块**：如果感知到闪光，抓取金块并返回出口
2. **探索安全方格**：规划前往最近的未访问安全方格
3. **射击wumpus**：如果有箭，尝试射击可能的wumpus位置
4. **冒险探索**：如果没有安全方格，冒险前往未证明不安全的方格
5. **撤退**：如果以上都失败，返回出口

### 3.6 逻辑状态估计

#### 3.6.1 精确状态估计

精确的信念状态是逻辑公式，表示与所有感知一致的可能状态集合。

**问题**：对于时刻$t$有$n$个流符号，信念状态集合有$2^{2^n}$个可能，精确表示可能需要指数级大小的公式。

#### 3.6.2 近似状态估计

**1-CNF近似**：用文字的合取式表示信念状态。

**方法**：
1. 对于每个符号$X^t$，尝试证明$X^t$和$¬X^t$
2. 可证明的文字加入新的信念状态
3. 丢弃先前的信念状态

**性质**：
- 保守近似：包含所有确实可能的状态
- 可能损失信息：某些可证明的事实可能无法单独证明
- 更新时间：与$t$无关（常数时间）

### 3.7 SATPlan规划

#### 3.7.1 基本思想

将规划问题编码为可满足性问题：

1. 构建包含初始状态、转移公理、目标断言的语句
2. 使用SAT求解器寻找满足赋值
3. 从模型中提取动作序列

#### 3.7.2 编码要素

**初始状态**（$t=0$）：
$$L_{1,1}^0 \land FacingEast^0 \land HaveArrow^0$$

**后继状态公理**（每个$t$）：
$$HaveArrow^{t+1} \Leftrightarrow (HaveArrow^t \land ¬Shoot^t)$$

**目标**（在$t$时刻）：
$$L_{2,3}^t \land HoldingGold^t$$

**前提公理**：
$$Shoot^t \Rightarrow HaveArrow^t$$

**动作排除公理**：
$$¬Forward^t \lor ¬Shoot^t$$

#### 3.7.3 算法

```
function SATPLAN(init, transition, goal, T_max) returns 解或failure
    for t = 0 to T_max do
        cnf ← TRANSLATE-TO-SAT(init, transition, goal, t)
        model ← SAT-SOLVER(cnf)
        if model非空 then
            return EXTRACT-SOLUTION(model)
    return failure
```

## 4. 定理与证明

### 4.1 后继状态公理的充分性

**定理**：给定完整的后继状态公理集、初始状态公理、前提公理和动作排除公理，SATPlan找到的模型对应于原始规划问题的有效解。

**证明概要**：

**可靠性**：
- 后继状态公理确保流的变化符合动作效果
- 前提公理确保动作只在前提满足时执行
- 动作排除公理确保没有冲突动作同时执行
- 因此，提取的动作序列是有效的规划

**完备性**：
- 任何有效的规划都可以表示为命题变量的赋值
- 该赋值满足所有公理
- 因此SAT求解器能够找到它

### 4.2 1-CNF近似的保守性

**定理**：1-CNF信念状态是精确信念状态的保守近似。

**证明**：

设$\phi$是精确信念状态，$\psi$是1-CNF近似。

**构造**：$\psi$包含所有可以从$\phi$单独证明的文字。

**包含关系**：
- 对于任何模型$m \models \phi$，$m$满足$\phi$中的所有文字
- 这些文字都在$\psi$中
- 因此$m \models \psi$
- 所以$M(\phi) \subseteq M(\psi)$

**保守性**：$\psi$包含所有与感知历史一致的状态（可能还包含额外状态）。

### 5.1 wumpus世界推理示例

**感知和动作序列**：

| 时刻 | 感知 | 动作 |

| 0 | $¬Stench^0 ∧ ¬Breeze^0 ∧ ¬Glitter^0 ∧ ¬Bump^0 ∧ ¬Scream^0$ | $Forward^0$ |
| 1 | $¬Stench^1 ∧ Breeze^1 ∧ ¬Glitter^1 ∧ ¬Bump^1 ∧ ¬Scream^1$ | $TurnRight^1$ |
| 2 | $¬Stench^2 ∧ Breeze^2 ∧ ¬Glitter^2 ∧ ¬Bump^2 ∧ ¬Scream^2$ | $TurnRight^2$ |
| 3 | $¬Stench^3 ∧ Breeze^3 ∧ ¬Glitter^3 ∧ ¬Bump^3 ∧ ¬Scream^3$ | $Forward^3$ |
| 4 | $¬Stench^4 ∧ ¬Breeze^4 ∧ ¬Glitter^4 ∧ ¬Bump^4 ∧ ¬Scream^4$ | $TurnRight^4$ |
| 5 | $¬Stench^5 ∧ ¬Breeze^5 ∧ ¬Glitter^5 ∧ ¬Bump^5 ∧ ¬Scream^5$ | $Forward^5$ |
| 6 | $Stench^6 ∧ ¬Breeze^6 ∧ ¬Glitter^6 ∧ ¬Bump^6 ∧ ¬Scream^6$ | - |

**推理结果**（在$t=6$时）：
- $ASK(KB, L_{1,2}^6) = true$：智能体知道它在$[1,2]$
- $ASK(KB, W_{1,3}) = true$：智能体找到了wumpus
- $ASK(KB, P_{3,1}) = true$：智能体找到了无底洞
- $ASK(KB, OK_{2,2}^6) = true$：$[2,2]$是安全的

### 5.2 SATPlan示例

**问题**：从$[1,1]$到达$[2,1]$的最短规划。

**编码**（$t=1$）：

初始状态：
$$L_{1,1}^0 \land FacingEast^0$$

目标：
$$L_{2,1}^1$$

后继状态公理：
$$L_{2,1}^1 \Leftrightarrow (L_{1,1}^0 \land FacingEast^0 \land Forward^0)$$

**求解**：
- SAT求解器找到模型：$Forward^0 = true$
- 提取规划：$[Forward]$

**验证**：执行$Forward$后，智能体确实在$[2,1]$。

### 5.3 混合智能体决策示例

**场景**：智能体在$[1,1]$，感知到闪光。

**决策过程**：

1. **检测闪光**：$ASK(KB, Glitter^t) = true$
2. **创建规划**：
   ```
   plan ← [Grab] + PLAN-ROUTE([1,1], [1,1], safe) + [Climb]
        ← [Grab, Climb]
   ```
3. **执行**：抓取金块，然后爬出洞穴

**性能**：获得$+1000$分（抓取金块并逃出），减去动作成本。

## 6. 一句话本质

**基于命题逻辑的智能体通过时序标记的流、后继状态公理和混合推理架构，将知识表示、逻辑推理与问题求解搜索相结合，实现了对动态世界的状态估计、安全决策和自动规划，其中SATPlan将规划问题编码为可满足性问题，利用现代SAT求解器高效生成最优动作序列。**

## 7. 总结与反思

### 7.1 关键要点

1. **时序表示**：使用时间上标标记流和感知，避免新旧信息矛盾。

2. **后继状态公理**：解决了框架问题，将转移模型大小从$O(mn)$减少到$O(mk)$。

3. **混合智能体**：结合逻辑推理（安全检测）和问题求解搜索（路径规划）。

4. **状态估计**：精确状态估计需要指数级空间，1-CNF近似提供保守的常数时间更新。

5. **SATPlan**：将规划问题编码为SAT问题，利用现代求解器高效求解。

### 7.2 常见误解对照表

| 常见误解 | 正确理解 |
|---------|---------|
| 效应公理足以描述动作 | 还需要框架公理或后继状态公理来描述未改变的状态 |
| SATPlan可以处理部分可观测环境 | SATPlan需要完整的初始状态，不能直接处理部分可观测性 |
| 1-CNF近似总是精确的 | 1-CNF是保守近似，可能损失信息 |
| 混合智能体只使用逻辑推理 | 混合智能体结合逻辑推理和搜索算法 |
| 后继状态公理消除了所有框架问题 | 还有推断框架问题（计算效率）和资格问题 |

### 7.3 反思问题

1. **框架问题的本质**：框架问题为什么重要？后继状态公理如何解决这个问题？还有什么相关问题（如资格问题）需要考虑？

2. **近似与精确**：在什么情况下1-CNF近似会损失重要信息？如何改进近似方法？

3. **可扩展性**：基于命题逻辑的智能体在扩展到大规模问题时面临什么挑战？一阶逻辑如何帮助解决这些问题？

4. **混合架构**：混合智能体中逻辑推理和搜索各扮演什么角色？这种分工为什么是有效的？
