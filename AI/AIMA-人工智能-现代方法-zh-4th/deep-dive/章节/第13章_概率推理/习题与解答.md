## 基础练习题

### 习题13.1 【难度：⭐】贝叶斯网络基础

**知识点**：贝叶斯网络结构、父节点、CPT

**题目**：
考虑以下贝叶斯网络：

```
Cloudy
  ├──► Sprinkler
  └──► Rain
         └──► WetGrass
Sprinkler ───► WetGrass
```

**问题**：
1. 列出所有变量及其父节点。
2. 如果要指定这个网络的CPT，总共需要多少个参数？（假设都是布尔变量）
3. 如果改用完全联合分布表示，需要多少参数？

**逐步提示**：
- 父节点是直接指向该变量的节点
- CPT条目数 = 2^(父节点数)，每个变量一个CPT
- 完全联合分布参数 = 2^n - 1

**详细解答**：

1. **变量和父节点**：

| 变量 | 父节点 | CPT条目数 |
|-----|--------|----------|
| Cloudy | ∅ | 2⁰ = 1 |
| Sprinkler | Cloudy | 2¹ = 2 |
| Rain | Cloudy | 2¹ = 2 |
| WetGrass | Sprinkler, Rain | 2² = 4 |

2. **贝叶斯网络参数总数**：
   1 + 2 + 2 + 4 = **9个参数**

3. **完全联合分布参数**：
   4个布尔变量：2⁴ - 1 = **15个参数**

**参数节省**：15 - 9 = 6（节省40%）

**考点解析**：
- 贝叶斯网络利用条件独立性紧凑表示联合分布
- 网络越稀疏（每个节点的父节点越少），节省越明显
- 对于n个变量，每个最多k个父节点：参数数量O(n × 2^k) vs O(2^n)

### 习题13.2 【难度：⭐】联合概率计算

**知识点**：链式法则、概率分解

**题目**：
使用入室盗窃网络的CPT（见教材图13-2），计算以下联合概率：

**CPT摘要**：
- P(Burglary) = 0.001
- P(Earthquake) = 0.002
- P(Alarm | B, E) = 0.95; P(Alarm | B, ¬E) = 0.94; P(Alarm | ¬B, E) = 0.29; P(Alarm | ¬B, ¬E) = 0.001
- P(JohnCalls | Alarm) = 0.90; P(JohnCalls | ¬Alarm) = 0.05
- P(MaryCalls | Alarm) = 0.70; P(MaryCalls | ¬Alarm) = 0.01

**问题**：
计算P(j, m, a, ¬b, ¬e)，即约翰和玛丽都打电话、警报响起、但没有入室盗窃和地震的概率。

**逐步提示**：
- 使用链式法则分解：P(j,m,a,¬b,¬e) = P(j|a) × P(m|a) × P(a|¬b,¬e) × P(¬b) × P(¬e)
- 从CPT中查找每个概率值
- 相乘得到结果

**详细解答**：

根据贝叶斯网络的联合分布分解：

```
P(j, m, a, ¬b, ¬e) 
= P(j | a) × P(m | a) × P(a | ¬b, ¬e) × P(¬b) × P(¬e)
= 0.90 × 0.70 × 0.001 × 0.999 × 0.998
= 0.90 × 0.70 × 0.001 × 0.997002
= 0.000628
```

**答案**：约 **0.000628** 或 **6.28 × 10⁻⁴**

**考点解析**：
- 贝叶斯网络的语义：联合分布 = 局部条件分布的乘积
- 给定完整赋值，概率计算是直接的条件概率相乘
- 注意补事件的概率：P(¬b) = 1 - P(b)

### 习题13.3 【难度：⭐⭐】d-分离判断

**知识点**：d-分离、条件独立性、道德图

**题目**：
考虑以下贝叶斯网络：

```
A ──► B ──► C
     ▲
     │
D ───┘
```

**问题**：
1. 判断以下独立性声明是否成立：
   - A ⊥ C | B
   - A ⊥ D | B
   - A ⊥ D | ∅
   - A ⊥ C | D

2. 对于每个成立的声明，解释原因。

**逐步提示**：
- 给定B时，A和C在链式结构中被阻塞
- 给定B时，A和D在分叉结构中被阻塞
- 使用道德图方法验证

**详细解答**：

1. **独立性判断**：

| 声明 | 是否成立 | 原因 |
|-----|---------|------|
| A ⊥ C \| B | ✓ | 链式结构，给定B阻塞路径 |
| A ⊥ D \| B | ✓ | 分叉结构（B是共同子节点），给定B阻塞 |
| A ⊥ D \| ∅ | ✗ | 存在路径 A→B←D，未给定B时C不阻塞 |
| A ⊥ C \| D | ✗ | D不是A-C路径上的节点，不影响A-C依赖 |

2. **详细解释**：

**A ⊥ C | B（链式结构阻塞）**：
- 路径：A → B → C
- 给定B，该路径被阻塞
- 因此A和C条件独立

**A ⊥ D | B（对撞结构）**：
- 路径：A → B ← D
- B是对撞节点
- 给定B（或其后代）会"打开"对撞路径
- 等等，这实际上会使A和D相关！

**修正**：
实际上：
- **A ⊥ D | ∅**：路径A→B←D，B未给定，路径自然阻塞 ✓
- **A ⊥ D | B**：给定B，对撞路径被"打开"，A和D相关 ✗

重新判断：

| 声明 | 是否成立 |
|-----|---------|
| A ⊥ C \| B | ✓ |
| A ⊥ D \| B | ✗（给定B使A和D相关） |
| A ⊥ D \| ∅ | ✓ |
| A ⊥ C \| D | ✓（D在并行路径上，给定D不影响A-C） |

**考点解析**：
- 链式（X→Y→Z）：给定Y阻塞路径
- 分叉（X←Y→Z）：给定Y阻塞路径
- 对撞（X→Y←Z）：未给定Y时路径阻塞，给定Y时路径打开
- d-分离是判断条件独立性的图形化方法

### 习题13.4 【难度：⭐⭐】马尔可夫毯

**知识点**：马尔可夫毯、条件独立性

**题目**：
在入室盗窃网络中：
```
Burglary ──┐
           ├──► Alarm ──► JohnCalls
Earthquake─┘           │
                       └──► MaryCalls
```

**问题**：
1. 定义马尔可夫毯的概念。
2. 找出每个变量的马尔可夫毯。
3. 验证：给定马尔可夫毯，变量独立于网络中所有其他节点。

**逐步提示**：
- 马尔可夫毯 = 父节点 + 子节点 + 子节点的其他父节点
- 检查每个变量的这三类节点
- 使用d-分离验证独立性

**详细解答**：

1. **马尔可夫毯定义**：
   节点X的马尔可夫毯MB(X) = Parents(X) ∪ Children(X) ∪ Parents(Children(X))
   
   给定MB(X)，X条件独立于网络中所有其他节点。

2. **各变量的马尔可夫毯**：

| 变量 | 父节点 | 子节点 | 子节点的其他父节点 | 马尔可夫毯 |
|-----|--------|--------|------------------|-----------|
| Burglary | ∅ | Alarm | Earthquake | {Earthquake, Alarm} |
| Earthquake | ∅ | Alarm | Burglary | {Burglary, Alarm} |
| Alarm | Burglary, Earthquake | JohnCalls, MaryCalls | ∅ | {Burglary, Earthquake, JohnCalls, MaryCalls} |
| JohnCalls | Alarm | ∅ | ∅ | {Alarm} |
| MaryCalls | Alarm | ∅ | ∅ | {Alarm} |

3. **验证示例（Burglary）**：
   - MB(Burglary) = {Earthquake, Alarm}
   - 给定Earthquake和Alarm，验证Burglary ⊥ JohnCalls
   - 路径：Burglary → Alarm → JohnCalls
   - 给定Alarm，路径被阻塞 ✓
   - 同理验证Burglary ⊥ MaryCalls ✓

**考点解析**：
- 马尔可夫毯是最小条件集合，使节点独立于其余网络
- 吉布斯采样只使用马尔可夫毯进行局部采样
- 马尔可夫毯性质使分布式推断成为可能

### 习题13.5 【难度：⭐⭐】枚举推断

**知识点**：精确推断、求和消元、证据处理

**题目**：
使用入室盗窃网络，计算P(Burglary | JohnCalls=true, MaryCalls=true)。

**简化CPT**：
- P(B) = 0.001, P(E) = 0.002
- P(A|B,E) = 0.95; P(A|B,¬E) = 0.94; P(A|¬B,E) = 0.29; P(A|¬B,¬E) = 0.001
- P(J|A) = 0.90, P(J|¬A) = 0.05
- P(M|A) = 0.70, P(M|¬A) = 0.01

**问题**：
1. 写出枚举推断的公式（对所有隐变量求和）。
2. 计算P(b | j, m)和P(¬b | j, m)。
3. 归一化得到最终答案。

**逐步提示**：
- 隐变量：Earthquake, Alarm
- 枚举公式：P(B|j,m) ∝ Σₑ Σₐ P(B)P(e)P(a|B,e)P(j|a)P(m|a)
- 将求和向内移动优化计算

**详细解答**：

1. **枚举公式**：
```
P(B | j, m) = α Σₑ Σₐ P(B) P(e) P(a | B, e) P(j | a) P(m | a)
            = α P(B) Σₑ P(e) Σₐ P(a | B, e) P(j | a) P(m | a)
```

2. **计算P(b | j, m)**：

```
P(b | j, m) ∝ P(b) Σₑ P(e) Σₐ P(a | b, e) P(j | a) P(m | a)

对于e=true：
Σₐ P(a | b, e) P(j | a) P(m | a)
= P(a | b, e)P(j | a)P(m | a) + P(¬a | b, e)P(j | ¬a)P(m | ¬a)
= 0.95 × 0.90 × 0.70 + 0.05 × 0.05 × 0.01
= 0.5985 + 0.000025
= 0.598525

对于e=false：
= 0.94 × 0.90 × 0.70 + 0.06 × 0.05 × 0.01
= 0.5922 + 0.00003
= 0.59223

求和：
P(b | j, m) ∝ 0.001 × (0.002 × 0.598525 + 0.998 × 0.59223)
           ∝ 0.001 × (0.001197 + 0.591)
           ∝ 0.001 × 0.5922
           ∝ 0.0005922
```

3. **计算P(¬b | j, m)**（类似计算）：
```
P(¬b | j, m) ∝ 0.999 × (0.002 × [0.29×0.9×0.7 + 0.71×0.05×0.01] 
                       + 0.998 × [0.001×0.9×0.7 + 0.999×0.05×0.01])
            ∝ 0.999 × (0.002 × 0.1827 + 0.998 × 0.00113)
            ∝ 0.999 × 0.00149
            ∝ 0.001489
```

4. **归一化**：
```
α = 1 / (0.0005922 + 0.001489) = 1 / 0.002081 ≈ 480.5

P(b | j, m) ≈ 0.0005922 × 480.5 ≈ 0.284
P(¬b | j, m) ≈ 0.001489 × 480.5 ≈ 0.716
```

**答案**：P(Burglary | j, m) ≈ **⟨0.284, 0.716⟩**

**考点解析**：
- 枚举推断直接利用联合分布定义
- 通过将求和向内移动避免重复计算
- 复杂度O(2^n)对n个布尔隐变量
- 实际中变量消元算法更高效

### 习题13.6 【难度：⭐⭐⭐】变量消元算法

**知识点**：因子、逐点乘积、求和消元、变量排序

**题目**：
使用变量消元计算P(Burglary | JohnCalls=true, MaryCalls=true)。

**因子定义**：
- f₁(B) = P(B) = ⟨0.001, 0.999⟩
- f₂(E) = P(E) = ⟨0.002, 0.998⟩
- f₃(A,B,E) = P(A|B,E)：2×2×2矩阵
- f₄(A) = P(J|A) = ⟨0.90, 0.05⟩（证据J=true已处理）
- f₅(A) = P(M|A) = ⟨0.70, 0.01⟩（证据M=true已处理）

**问题**：
1. 按消元顺序A, E执行变量消元，写出每一步的因子操作。
2. 计算最终结果。
3. 如果改按E, A顺序消元，中间因子f₆的大小有何不同？

**逐步提示**：
- 变量消元：从最内层求和开始
- 逐点乘积：合并相关因子
- 求和消元：对变量所有值求和

**详细解答**：

1. **消元顺序A, E**：

**步骤1：消元A**
```
f₆(B, E) = Σₐ f₃(A, B, E) × f₄(A) × f₅(A)

对于每个(B, E)组合：
- 当B=b, E=e:
  f₆(b,e) = f₃(a,b,e)×f₄(a)×f₅(a) + f₃(¬a,b,e)×f₄(¬a)×f₅(¬a)
          = 0.95×0.9×0.7 + 0.05×0.05×0.01
          = 0.5985 + 0.000025 = 0.598525

- 当B=b, E=¬e: 0.59223（见上题计算）
- 当B=¬b, E=e: 0.1827
- 当B=¬b, E=¬e: 0.00113

f₆是2×2矩阵：
       E=e    E=¬e
B=b  0.5985  0.5922
B=¬b 0.1827  0.0011
```

**步骤2：消元E**
```
f₇(B) = Σₑ f₂(E) × f₆(B, E)

f₇(b) = f₂(e)×f₆(b,e) + f₂(¬e)×f₆(b,¬e)
      = 0.002×0.5985 + 0.998×0.5922
      = 0.001197 + 0.591016
      = 0.592213

f₇(¬b) = 0.002×0.1827 + 0.998×0.0011
       = 0.000365 + 0.001098
       = 0.001463
```

**步骤3：与f₁相乘并归一化**
```
P(B | j, m) = α f₁(B) × f₇(B)
            = α ⟨0.001×0.592213, 0.999×0.001463⟩
            = α ⟨0.000592, 0.001462⟩
            = ⟨0.288, 0.712⟩
```

（与枚举结果略有差异来自四舍五入）

2. **消元顺序E, A**：
- 先消元E：f₆(A, B) = Σₑ f₂(E) × f₃(A, B, E)
  产生2×2矩阵（A和B各2值）
- 再消元A：f₇(B) = Σₐ f₆(A, B) × f₄(A) × f₅(A)

**关键区别**：
- 顺序A, E：中间因子f₆是2×2（B, E）
- 顺序E, A：中间因子f₆是2×2（A, B）

在这个例子中大小相同，但一般情况下不同。

**最优排序**：选择使中间因子最小的变量先消元。

**考点解析**：
- 变量消元是动态规划在概率推断中的应用
- 消元顺序严重影响计算复杂度
- 最优排序是NP困难问题，常用启发式
- 因子大小与变量数量呈指数关系

### 习题13.7 【难度：⭐⭐⭐】似然加权采样

**知识点**：近似推断、似然加权、样本权重

**题目**：
使用似然加权估计P(Rain | Cloudy=true, WetGrass=true)。

**网络结构**：
```
Cloudy ──► Sprinkler
     │
     └──► Rain ──► WetGrass
           ▲
           │
     Sprinkler
```

**CPT**：
- P(Cloudy) = 0.5
- P(Sprinkler | Cloudy) = 0.1, P(Sprinkler | ¬Cloudy) = 0.5
- P(Rain | Cloudy) = 0.8, P(Rain | ¬Cloudy) = 0.2
- P(WetGrass | Sprinkler, Rain)：
  - P(W | s, r) = 0.99, P(W | s, ¬r) = 0.90
  - P(W | ¬s, r) = 0.90, P(W | ¬s, ¬r) = 0.00

**证据**：Cloudy=true, WetGrass=true

**问题**：
1. 说明似然加权的采样过程。
2. 生成一个样本，计算其权重。
3. 解释权重计算公式的含义。

**逐步提示**：
- 证据变量固定：Cloudy=true, WetGrass=true
- 非证据变量采样：Sprinkler, Rain
- 权重 = 证据变量条件概率的乘积

**详细解答**：

1. **似然加权采样过程**：

**步骤**：
- 初始化权重w = 1.0
- 按拓扑顺序处理每个变量：
  * 如果是证据变量：将权重乘以该证据值的概率
  * 如果是非证据变量：从其条件分布中采样

2. **生成样本示例**：

**Cloudy**（证据，值为true）：
- w = w × P(Cloudy=true) = 1.0 × 0.5 = 0.5

**Sprinkler**（非证据）：
- 从P(Sprinkler | Cloudy=true) = ⟨0.1, 0.9⟩采样
- 假设采样结果：Sprinkler = false

**Rain**（非证据）：
- 从P(Rain | Cloudy=true) = ⟨0.8, 0.2⟩采样
- 假设采样结果：Rain = true

**WetGrass**（证据，值为true）：
- w = w × P(WetGrass=true | Sprinkler=false, Rain=true)
- w = 0.5 × 0.90 = 0.45

**样本**：[Cloudy=true, Sprinkler=false, Rain=true, WetGrass=true]
**权重**：0.45

3. **权重公式含义**：

权重 = ∏_{证据变量Eᵢ} P(Eᵢ = eᵢ | Parents(Eᵢ))

- 权重表示该样本与证据的**一致程度**
- 如果采样过程中"恰好"生成与证据一致的值，概率就是权重
- 权重修正了采样分布与目标后验分布之间的差异
- 权重越大，样本对后验估计的贡献越大

**后验估计**：
生成N个样本后：
```
P(Rain=true | evidence) ≈ Σ_{Rain=true的样本} wᵢ / Σ_{所有样本} wᵢ
```

**考点解析**：
- 似然加权解决拒绝采样效率低的问题
- 所有样本都与证据一致，没有浪费
- 权重补偿了采样分布与目标分布的差异
- 但权重可能差异很大，少数样本主导估计

### 习题13.8 【难度：⭐⭐⭐】吉布斯采样

**知识点**：MCMC、吉布斯采样、马尔可夫毯、平稳分布

**题目**：
考虑简化网络：Cloudy → Rain，证据Rain=true。

**CPT**：
- P(Cloudy) = 0.5
- P(Rain | Cloudy) = 0.8, P(Rain | ¬Cloudy) = 0.2

**问题**：
1. 写出吉布斯采样的马尔可夫链状态空间。
2. 计算P(Cloudy | mb(Cloudy))，即给定马尔可夫毯时Cloudy的采样分布。
3. 从初始状态Cloudy=false开始，模拟两步吉布斯采样。

**逐步提示**：
- 证据变量Rain=true固定不变
- 只有Cloudy需要采样
- 马尔可夫毯计算使用公式P(xᵢ|mb) ∝ P(xᵢ|parents) × ∏ P(children|parents)

**详细解答**：

1. **状态空间**：
   - 证据固定：Rain = true
   - 只有Cloudy变化：{true, false}
   - 状态空间：{Cloudy=true, Cloudy=false}

2. **采样分布P(Cloudy | mb(Cloudy))**：

MB(Cloudy) = Parents(Cloudy) ∪ Children(Cloudy) ∪ Parents(Children(Cloudy))
           = ∅ ∪ {Rain} ∪ ∅
           = {Rain}

给定Rain=true，计算：

```
P(Cloudy=true | Rain=true) 
∝ P(Cloudy=true) × P(Rain=true | Cloudy=true)
∝ 0.5 × 0.8 = 0.40

P(Cloudy=false | Rain=true)
∝ P(Cloudy=false) × P(Rain=true | Cloudy=false)
∝ 0.5 × 0.2 = 0.10

归一化：
P(Cloudy=true | Rain=true) = 0.40 / 0.50 = 0.8
P(Cloudy=false | Rain=true) = 0.10 / 0.50 = 0.2
```

3. **模拟采样**：

**初始状态**：Cloudy = false

**第1步**：
- 从P(Cloudy | Rain=true) = ⟨0.8, 0.2⟩采样
- 假设采样结果：Cloudy = true
- 新状态：Cloudy = true

**第2步**：
- 从P(Cloudy | Rain=true) = ⟨0.8, 0.2⟩采样
- 假设采样结果：Cloudy = false
- 新状态：Cloudy = false

**状态序列**：false → true → false → ...

经过足够多步后，统计Cloudy=true的比例，应接近0.8（真实的后验概率）。

**验证**：
用贝叶斯法则直接计算：
```
P(Cloudy=true | Rain=true) = P(Rain|Cloudy)P(Cloudy) / P(Rain)
= 0.8 × 0.5 / (0.8×0.5 + 0.2×0.5)
= 0.4 / 0.5 = 0.8 ✓
```

**考点解析**：
- 吉布斯采样在变量值空间随机游走
- 每次只改变一个变量，以其马尔可夫毯为条件
- 满足细致平衡，平稳分布就是目标后验
- 实际应用中需要"老化期"（burn-in）和大量样本

### 习题13.9 【难度：⭐⭐⭐】网络结构学习

**知识点**：节点排序、网络复杂性、因果建模

**题目**：
考虑入室盗窃网络的五个变量：Burglary(B), Earthquake(E), Alarm(A), JohnCalls(J), MaryCalls(M)。

**问题**：
1. 按因果顺序B, E, A, J, M构建网络（图13-2），需要多少参数？
2. 按逆序M, J, A, E, B构建网络（图13-3b），需要多少参数？
3. 解释为什么因果顺序产生的网络更紧凑。

**逐步提示**：
- 按顺序逐个添加变量，确定最小父节点集
- 参数数 = Σ 2^{父节点数}
- 因果顺序通常更稀疏

**详细解答**：

1. **因果顺序B, E, A, J, M**：

| 顺序 | 变量 | 父节点选择 | 父节点数 | CPT条目 |
|-----|------|-----------|---------|--------|
| 1 | B | 无 | 0 | 1 |
| 2 | E | 无（B与E独立） | 0 | 1 |
| 3 | A | B, E（A的直接原因） | 2 | 4 |
| 4 | J | A（J的直接原因） | 1 | 2 |
| 5 | M | A（M的直接原因） | 1 | 2 |

**总计**：1 + 1 + 4 + 2 + 2 = **10个参数**

2. **逆序M, J, A, E, B**：

| 顺序 | 变量 | 父节点选择 | 父节点数 | CPT条目 |
|-----|------|-----------|---------|--------|
| 1 | M | 无 | 0 | 1 |
| 2 | J | M（M给J提供了A的信息） | 1 | 2 |
| 3 | A | M, J（两者都提供信息） | 2 | 4 |
| 4 | E | A, B? | ? | ... |

详细分析（见图13-3b）：
- M: 无父节点（1参数）
- J: 需要M作为父节点（2参数）
- A: 需要M和J作为父节点（4参数）
- E: 需要A（E通过A影响M,J）...实际上根据条件独立性，只需要A
- B: 需要A和E（4参数）

**总计**：约**31个参数**

3. **因果顺序的优势**：

**因果顺序**：
- 每个变量只需要其直接原因作为父节点
- 反映了真实的数据生成过程
- 网络稀疏，参数少

**非因果顺序**：
- 后期变量（如M）没有"原因"可依赖
- 需要依赖大量"诊断"信息
- 父节点多，网络稠密
- 参数估计困难（需要回答"给定两个人打电话，地震概率是多少"）

**关键洞察**：
- 因果模型回答"为什么"的问题更自然
- P(症状|疾病)比P(疾病|症状)更容易估计
- 因果模型对干预和变化更鲁棒

**考点解析**：
- 节点排序影响网络复杂性和自然性
- 因果顺序通常更优
- 但任何顺序都可以正确表示联合分布
- 这是因果网络理论发展的动机之一

### 习题13.10 【难度：⭐⭐⭐⭐】因果推断与后门准则

**知识点**：因果网络、do-算子、后门准则、混杂变量

**题目**：
考虑研究吸烟(Smoking)对肺癌(LungCancer)的影响：

```
GeneticFactor ──► Smoking ──► LungCancer
           │              ▲
           │              │
           └──► Tar └───►─┘
```

**问题**：
1. 解释为什么直接计算P(LungCancer | Smoking)不能给出因果效应。
2. 识别从Smoking到LungCancer的后门路径。
3. 使用后门准则找到合适的调整集合。
4. 写出计算P(LungCancer | do(Smoking))的公式。

**逐步提示**：
- 混杂变量：同时影响原因和结果的变量
- 后门路径：以箭头指向原因开始的路径
- 后门准则：集合Z阻塞所有后门路径，且不包含后代

**详细解答**：

1. **P(LungCancer | Smoking) ≠ 因果效应**：

P(LungCancer | Smoking)包含两条路径的影响：
- **因果路径**：Smoking → LungCancer（我们关心的）
- **混杂路径**：Smoking ← GeneticFactor → ... → LungCancer（伪相关）

观察到的相关性可能部分由共同原因GeneticFactor引起，而非真正的因果关系。

2. **后门路径**：

后门路径是以箭头指向Smoking开始的路径：
- Smoking ← GeneticFactor → ... → LungCancer

这条路径通过共同原因GeneticFactor产生伪相关。

3. **后门准则应用**：

**后门准则**：找到集合Z使得：
- Z阻塞所有从Smoking到LungCancer的后门路径
- Z不包含Smoking的后代

**候选集合**：
- {GeneticFactor}：阻塞后门路径，不是后代 ✓
- {Tar}：Tar是Smoking的后代，违反条件 ✗
- ∅：不阻塞后门路径 ✗

**合适的调整集合**：Z = {GeneticFactor}

4. **调整公式**：

```
P(LC | do(S)) = Σ_g P(LC | S, GeneticFactor=g) × P(GeneticFactor=g)
```

其中LC=LungCancer, S=Smoking。

**解释**：
- 按GeneticFactor分层
- 在每层内计算条件概率
- 用GeneticFactor的分布加权平均
- 这消除了混杂因素的影响

**实际应用**：
- 随机对照试验（RCT）：随机分配Smoking，消除混杂
- 观察性研究：使用后门准则统计调整

**考点解析**：
- 关联≠因果是统计学的核心问题
- 因果网络提供了形式化定义因果效应的框架
- 后门准则是观察性研究中估计因果效应的重要工具
- 这是朱迪亚·珀尔因果推断理论的核心贡献

### 习题13.11 【难度：⭐⭐⭐⭐】推断复杂性分析

**知识点**：计算复杂性、NP困难、#P困难、归约

**题目**：
证明贝叶斯网络中的精确推断是NP困难的。

**问题**：
1. 将3-SAT问题归约到贝叶斯网络推断。
2. 说明归约的正确性。
3. 解释为什么精确推断实际上是#P困难的。

**逐步提示**：
- 3-SAT：给定CNF公式，判断是否存在满足赋值
- 构建贝叶斯网络，使可满足性对应P(S=true) > 0
- #P：计数问题，计算满足赋值的数量

**详细解答**：

1. **从3-SAT到贝叶斯网络的归约**：

**给定**：3-CNF公式，例如 (X₁ ∨ ¬X₂ ∨ X₃) ∧ (¬X₁ ∨ X₂ ∨ X₄)

**构建网络**：

```
X₁ ──► C₁
¬X₂ ──►    S
X₃ ──► ───►

¬X₁ ──► C₂
X₂ ───►    
X₄ ──► ───►
```

**网络结构**：
- **第一层**：布尔变量X₁, X₂, ..., Xₙ作为根节点
  - P(Xᵢ=true) = 0.5（均匀分布）
  
- **第二层**：子句节点Cⱼ
  - 父节点：子句中的文字对应的变量（或其否定）
  - CPT：确定性OR，当且仅当父节点赋值满足子句时为true
  
- **第三层**：合取节点S
  - 父节点：所有子句节点C₁, C₂, ..., Cₘ
  - CPT：确定性AND，当且仅当所有子句为true时为true

2. **归约正确性**：

**定理**：P(S=true) > 0 当且仅当原3-SAT公式可满足。

**证明**：
- (⇒) 如果P(S=true) > 0，则存在某个赋值使S=true
  - 这意味着所有子句Cⱼ都为true
  - 即存在满足3-SAT公式的赋值
  
- (⇐) 如果3-SAT可满足，设v是满足赋值
  - P(变量取v的值) = (0.5)ⁿ > 0
  - 在该赋值下，所有Cⱼ=true，S=true
  - 因此P(S=true) > 0

**复杂度**：归约在多项式时间内完成。

**结论**：贝叶斯网络推断至少和3-SAT一样难，即**NP困难**。

3. **#P困难性**：

实际上可以证明更强结果：精确推断是**#P困难**的。

**#P类**：计数问题，"有多少个解"

**证明思路**：
- 可满足赋值的数量 = P(S=true) × 2ⁿ
- 如果能计算P(S=true)，就能计算满足赋值数量
- 计数满足赋值是#P完全的
- 因此贝叶斯网络推断是#P困难的

**#P严格难于NP**：
- NP：是否存在解？
- #P：有多少个解？
- 如果能解决#P，就能解决NP（但反之不成立）

**考点解析**：
- 复杂性分析揭示问题的本质难度
- 尽管精确推断是NP/#P困难的，实际中许多网络可处理
- 近似推断通常是多项式时间的
- 网络拓扑（树宽）决定实际复杂性

### 习题13.12 【难度：⭐⭐⭐⭐】混合贝叶斯网络

**知识点**：连续变量、线性高斯分布、概率单位模型

**题目**：
考虑以下混合贝叶斯网络：

```
Subsidy ──► Cost ──► Buys
Harvest ──►
```

**CPT**：
- Subsidy ∈ {true, false}，P(Subsidy) = 0.5
- Harvest ~ N(μ=100, σ²=100)（连续）
- Cost | Subsidy, Harvest：
  - Cost | subsidy, h ~ N(20 - 0.1h, 4)
  - Cost | ¬subsidy, h ~ N(30 - 0.1h, 9)
- Buys | Cost：使用概率单位模型
  - P(Buys | Cost=c) = Φ((15-c)/2)，其中Φ是标准正态CDF

**问题**：
1. 解释为什么Cost的分布是高斯的。
2. 给定Subsidy=true, Harvest=120，计算Cost的分布。
3. 在该条件下，计算P(Buys=true)。

**逐步提示**：
- 线性高斯：子节点均值是父节点的线性函数
- 正态分布的线性变换仍是正态
- 概率单位模型将连续值映射到概率

**详细解答**：

1. **Cost的高斯分布**：

网络只包含：
- 离散父节点的连续子节点（线性高斯）
- 连续父节点的离散子节点（概率单位）

对于Cost节点：
- 给定Subsidy和Harvest，Cost是高斯分布
- 这是**线性高斯条件分布**
- 均值：μ = a × Harvest + b（线性函数）
- 方差：常数（不依赖父节点值）

即使Subsidy未被观测，Cost的边际分布是**混合高斯**（两个高斯的加权和）。

2. **给定条件下的Cost分布**：

给定：Subsidy=true, Harvest=120

```
Cost | subsidy, h=120 ~ N(20 - 0.1×120, 4)
                     ~ N(20 - 12, 4)
                     ~ N(8, 4)
```

即：均值8，方差4，标准差2。

3. **计算P(Buys=true)**：

使用概率单位模型：
```
P(Buys | Cost=c) = Φ((15-c)/2)
```

但Cost是随机变量，需要积分：
```
P(Buys=true | subsidy, h=120) 
= ∫ P(Buys | Cost=c) × P(Cost=c | subsidy, h=120) dc
= ∫ Φ((15-c)/2) × N(c; 8, 4) dc
```

这个积分没有闭式解，需要数值计算。

**近似计算**（使用Cost的期望值）：
```
E[Cost] = 8
P(Buys | Cost=8) = Φ((15-8)/2) = Φ(3.5) ≈ 0.9998
```

**更精确的方法**：
- 采样：从Cost分布采样，计算平均概率
- 变分近似
- 消息传递算法（对于特殊结构）

**考点解析**：
- 混合网络结合了离散和连续变量
- 线性高斯允许精确推断（对于连续部分）
- 离散-连续连接通常需要近似
- 实际应用中常用采样或变分方法
