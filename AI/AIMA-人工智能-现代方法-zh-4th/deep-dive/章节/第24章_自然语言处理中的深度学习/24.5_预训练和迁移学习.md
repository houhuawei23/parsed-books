# 24.5 预训练和迁移学习

## 背景与动机

### 数据标注的挑战

传统监督学习需要大量标注数据：

| 任务 | 标注成本 | 示例 |
|------|----------|------|
| 图像分类 | 低 | 标注"猫"/"狗" |
| 词性标注 | 中 | 需要语言学知识 |
| 句法分析 | 高 | 需要构建树结构 |
| 机器翻译 | 很高 | 需要双语专家 |

**关键观察**：
- 互联网上有海量**未标注文本**（每天超过1000亿词）
- 人类学习语言时，大部分输入也是无标注的

### 迁移学习范式

**核心思想**：
1. **预训练（Pre-training）**：在大规模无标注数据上学习通用表示
2. **微调（Fine-tuning）**：在特定任务的小规模标注数据上调整

```
大规模无标注文本 → 预训练模型 → 下游任务微调 → 特定任务模型
        ↓               ↓              ↓
     学习语言规律    通用语言理解    任务特定优化
```

## 预训练词嵌入

### 从统计到预测

**第一代预训练**：Word2Vec, GloVe（24.1节）
- 静态词向量
- 上下文无关

**局限**：
```
"bank" 在 "river bank" 和 "bank account" 中有不同含义
但预训练词嵌入只给出一个向量
```

### GloVe详解

**共现概率**：

$$P_{ij} = \frac{X_{ij}}{X_i}$$

其中 $X_{ij}$ 是词 $i$ 和 $j$ 在窗口内共现次数，$X_i = \sum_j X_{ij}$。

**词向量约束**：

$$\mathbf{E}_i \cdot \mathbf{E}_j' = \log P_{ij}$$

词向量的点积等于共现概率的对数。

**实际应用**：
- 材料科学领域训练：可以回答"NiFe对铁磁性，IrMn对什么？"
- 答案："反铁磁性"

## 预训练上下文表示

### 动机：一词多义

**多义词问题**：
- "rose" 可以是花，也可以是动词"rise"的过去式
- "need" 在 "you need to see this" vs "humans need oxygen" 中语义不同

**解决方案**：上下文相关的词表示。

### ELMo：Embeddings from Language Models

**核心思想**：使用双向LSTM语言模型的隐藏状态作为词表示。

**架构**：
```
输入层 → LSTM层1 → LSTM层2 → 输出层
   ↓         ↓          ↓
词嵌入   句法信息   语义信息
```

**表示计算**：

$$\text{ELMo}_k = \gamma \sum_{j=0}^{L} s_j \mathbf{h}_{k,j}$$

其中：
- $\mathbf{h}_{k,j}$：第 $k$ 个词在第 $j$ 层的隐藏状态
- $s_j$：可学习的层权重
- $\gamma$：缩放因子

**特点**：
- 双向（使用两个LSTM，一个从左到右，一个从右到左）
- 深层表示（使用多层LSTM的加权和）
- 字符级CNN处理未登录词

## 掩码语言模型

### 标准语言模型的局限

**从左到右的预测**：
```
"The river ___ five feet"
```
仅看到左边的"The river"，难以确定空白处。

**从右到左的预测**：
同样存在信息不完整的问题。

### MLM：Masked Language Model

**核心思想**：
- 随机mask（遮盖）输入中的部分词
- 使用双向上下文预测被mask的词

**示例**：
```
原始句子：The river rose five feet
Mask后：  The river [MASK] five feet
目标：    预测 "rose"
```

**训练过程**：
1. 随机选择15%的token进行预测
2. 被选中的token：
   - 80%概率替换为[MASK]
   - 10%概率替换为随机词
   - 10%概率保持不变

**优势**：
- 真正的双向建模
- 无需标注数据
- 可学习深层次的语义关系

## BERT：Bidirectional Encoder Representations from Transformers

### 架构

**仅编码器Transformer**：
- 使用Transformer的Encoder部分
- 12层（Base）或24层（Large）
- 隐藏维度768（Base）或1024（Large）

### 预训练任务

**任务1：Masked LM**

如上所述的掩码语言模型。

**任务2：Next Sentence Prediction (NSP)**

输入两个句子，预测第二个句子是否是第一个的后续：

```
输入A："The man went to the store."
输入B："He bought a gallon of milk."
标签：IsNext ✓

输入A："The man went to the store."
输入B："The Eiffel Tower is in Paris."
标签：NotNext ✗
```

### 微调

**分类任务**：
```
[CLS] 输入文本 [SEP]
   ↓
取[CLS]位置的输出 → 全连接层 → Softmax → 类别
```

**序列标注**：
```
[CLS] 词1 词2 词3 ... [SEP]
   ↓
每个词的输出 → 全连接层 → Softmax → 标签
```

**问答任务**：
```
[CLS] 问题 [SEP] 段落 [SEP]
   ↓
输出两个位置：答案开始位置、答案结束位置
```

## GPT：Generative Pre-Training

### 架构差异

**仅解码器Transformer**：
- 使用Transformer的Decoder部分
- 单向（自回归）语言模型
- 适合生成任务

### 预训练

**标准语言建模**：

$$P(w_1, w_2, ..., w_n) = \prod_{i=1}^{n} P(w_i | w_1, ..., w_{i-1})$$

**目标**：给定上文，预测下一个词。

### 与BERT的对比

| 特性 | BERT | GPT |
|------|------|-----|
| 架构 | Encoder | Decoder |
| 方向 | 双向 | 单向 |
| 预训练任务 | MLM + NSP | 自回归LM |
| 适合任务 | 理解任务 | 生成任务 |
| 代表应用 | 分类、问答 | 文本生成、对话 |

## T5：Text-to-Text Transfer Transformer

### 统一框架

**核心思想**：将所有NLP任务统一为文本到文本的转换。

**任务格式**：
```
翻译：       "translate English to German: That is good."
摘要：       "summarize: The article content..."
问答：       "question: What is the capital of France? context: ..."
分类：       "cola sentence: The movie is good." → "acceptable"
```

### 架构

**完整的Encoder-Decoder**：
- Encoder：理解输入
- Decoder：生成输出

### 预训练

**Span Corruption**：
- 随机mask连续的span（而非单个token）
- 用唯一的哨兵token替换
- Decoder生成被mask的span

```
原始：Thank you for inviting me to the party last week.
Mask：Thank you <X> me to the party <Y> week.
目标：<X> for inviting <Y> last
```

```

```

## 微调策略

### 标准微调

1. 加载预训练权重
2. 添加任务特定的输出层
3. 在下游任务数据上训练所有参数

### 高效微调方法

**1. 特征提取（Feature Extraction）**：
- 冻结预训练模型
- 只训练输出层
- 适合数据极少的场景

**2. 逐层解冻（Layer-wise Unfreezing）**：
- 从顶层开始逐层解冻
- 平衡稳定性和适应性

**3. 学习率差异化**：
- 底层：较小学习率（保留通用知识）
- 上层：较大学习率（学习任务特定特征）

**4. Adapter Layers**：
- 在Transformer层中插入小型adapter模块
- 只训练adapter参数
- 大幅减少可训练参数

## 关键成功因素

### 为什么预训练有效？

1. **语言规律的普遍性**：
   - 语法、语义、语用规则跨任务通用

2. **表示学习的层次性**：
   - 底层：词法、句法
   - 中层：语义关系
   - 高层：推理、世界知识

3. **迁移学习的正则化效应**：
   - 预训练权重作为好的初始化点
   - 限制搜索空间，防止过拟合

### 规模效应

**数据规模**：更大的预训练数据 → 更好的泛化

**模型规模**：更多参数 → 更强的表达能力

**计算规模**：更多训练步数 → 更充分的优化

## 应用实例

### 阅读理解（SQuAD）

```
段落："The University of Notre Dame began late on..."
问题："When did the University of Notre Dame begin?"
答案："late on the bitterly cold afternoon of November 26, 1842"
```

**BERT/GPT效果**：接近或超过人类水平

### 情感分析

```
输入："This movie was a complete waste of time."
输出：负面（Negative）
```

### 命名实体识别

```
输入："Apple Inc. is planning to open a new store in Paris."
输出：Apple Inc. (ORG), Paris (LOC)
```

## 局限与挑战

### 计算资源需求

| 模型 | 参数量 | 训练成本 |
|------|--------|----------|
| BERT-Base | 110M | ~$7K |
| BERT-Large | 340M | ~$20K |
| GPT-3 | 175B | ~$4.6M |
| GPT-4 | 估计>1T | 极高 |

### 其他挑战

1. **上下文长度限制**：通常512或1024个token
2. **知识更新困难**：预训练知识可能过时
3. **可解释性不足**：黑盒模型
4. **偏见与公平性**：预训练数据中的偏见被继承

## 关键公式总结

| 公式 | 说明 |
|------|------|
| $\text{MLM Loss} = -\sum_{i \in \mathcal{M}} \log P(x_i \| \mathbf{x}_{\setminus \mathcal{M}})$ | 掩码语言模型损失 |
| $\text{NSP Loss} = -\log P(\text{IsNext} \| \text{sent}_A, \text{sent}_B)$ | 下一句预测损失 |
| $\text{LM Loss} = -\sum_t \log P(x_t \| x_{<t})$ | 自回归语言模型损失 |

## 小结

预训练-微调范式彻底改变了NLP：

1. **范式转变**：
   - 从任务特定架构到统一预训练模型
   - 从随机初始化到预训练权重初始化

2. **关键技术**：
   - 掩码语言模型（BERT）
   - 自回归语言模型（GPT）
   - Text-to-Text统一框架（T5）

3. **核心优势**：
   - 利用大规模无标注数据
   - 学习通用语言表示
   - 下游任务数据需求大幅降低

4. **发展趋势**：
   - 模型规模持续增长
     - 多模态预训练（文本+图像）
   - 高效微调方法研究
