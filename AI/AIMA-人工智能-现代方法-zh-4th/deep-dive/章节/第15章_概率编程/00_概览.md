## 三、各节核心要点

### 15.1 关系概率模型(RPM)

**核心思想**: 用一阶逻辑风格的语法定义模板，自动生成贝叶斯网络

```
┌────────────────────────────────────────────────────────────┐
│                    RPM 模型结构                             │
├────────────────────────────────────────────────────────────┤
│  类型定义: Customer, Book                                   │
│  函数符号: Honest(c), Kindness(c), Quality(b)               │
│           Recommendation(c,b)                               │
│  依赖声明: Honest(c) ~ <0.99, 0.01>                         │
│           Recommendation(c,b) ~ RecCPT(Honest(c),...        │
│                              Kindness(c), Quality(b))       │
├────────────────────────────────────────────────────────────┤
│  语义: 对给定常量集合，实例化为等价贝叶斯网络                 │
│  参数规模: O(1) 而非 O(n²)                                   │
└────────────────────────────────────────────────────────────┘
```

**关键技术**:
- **落地(Grounding)**: 将RPM展开为贝叶斯网络
- **提升推断(Lifted Inference)**: 避免完全落地，利用对称性
- **关系不确定性**: 函数值未知导致网络结构变化

### 15.2 开宇宙概率模型(OUPM)

**核心突破**: 放弃数据库语义，允许世界中对象数量不确定

```
┌────────────────────────────────────────────────────────────┐
│                   OUPM 核心机制                             │
├────────────────────────────────────────────────────────────┤
│  1. 数字语句(Number Statement)                              │
│     #Customer ~ UniformInt(1, 3)                           │
│     #LoginID(Owner=c) ~ if Honest(c) then Exactly(1)       │
│                          else UniformInt(2, 5)             │
│                                                             │
│  2. 起源函数(Origin Function)                               │
│     对象表示为生成历史: ⟨LoginID, ⟨Owner, ⟨Customer,2⟩⟩, 3⟩│
│                                                             │
│  3. 拓扑采样                                                │
│     按生成顺序: #Customer → #Book → Honest → ...          │
│     世界概率 = 各采样值概率的乘积                            │
└────────────────────────────────────────────────────────────┘
```

**关键不确定性类型**:
1. **存在不确定性**: 不知道有多少对象存在
2. **身份不确定性**: 不知道两个观测是否指向同一对象

**典型应用**:
- 引文匹配(CiteSeer、Google Scholar)
- 核条约监控(NET-VISA系统)
- 多目标跟踪

### 15.3 追踪复杂世界

**问题定义**: 多目标跟踪中的数据关联问题

```
┌────────────────────────────────────────────────────────────┐
│                 多目标跟踪模型                              │
├────────────────────────────────────────────────────────────┤
│  挑战:                                                       │
│    - 虚假警报(False alarms/Clutter)                        │
│    - 检测故障(Missed detections)                           │
│    - 目标进出(Target birth/death)                          │
│    - 观测-目标关联不确定                                     │
├────────────────────────────────────────────────────────────┤
│  推断方法:                                                   │
│    - 最近邻滤波: 贪婪匹配                                    │
│    - 匈牙利算法: 最优分配                                    │
│    - MCMC: 探索关联假设空间                                  │
│    - 粒子滤波: 多假设维护                                    │
│    - Rao-Blackwellization: 联合卡尔曼滤波                    │
└────────────────────────────────────────────────────────────┘
```

### 15.4 作为概率模型的程序

**核心洞察**: 任何带随机选择的程序都定义了一个概率分布

```
┌────────────────────────────────────────────────────────────┐
│              生成程序示例(OCR)                              │
├────────────────────────────────────────────────────────────┤
│  function GENERATE-IMAGE():                                │
│      letters ← GENERATE-LETTERS(10)                        │
│      return RENDER-NOISY-IMAGE(letters, 32, 128)           │
│                                                             │
│  function GENERATE-LETTERS(λ):                             │
│      n ~ Poisson(λ)                                        │
│      for i = 1 to n:                                       │
│          letters[i] ~ UniformChoice({a,b,c,...})           │
│      return letters                                        │
│                                                             │
│  // 执行轨迹 = 可能世界                                       │
│  // 轨迹概率 = 各随机选择概率的乘积                           │
└────────────────────────────────────────────────────────────┘
```

**推断方法**:
- **拒绝采样**: 运行程序，保留符合证据的轨迹
- **似然加权**: 跟踪轨迹权重
- **MCMC**: 采样和修改执行轨迹
- **推断编译**: 自动优化推断过程

## 四、核心算法与公式

### 4.1 RPM落地算法
```
输入: RPM模型，常量集合C
输出: 等价贝叶斯网络

for each 类型T的常量c:
    for each 依赖声明 f(x) ~ Distribution:
        实例化变量 f(c)
        根据依赖关系添加父节点
        设置条件概率表
```

### 4.2 OUPM拓扑采样
```
世界概率 = ∏ P(变量值 | 父变量值)

采样顺序:
1. 数字变量(无起源): #Customer, #Book
2. 基本随机变量: Honest(c), Kindness(c)
3. 有起源的数字变量: #LoginID(Owner=c)
4. 依赖起源的随机变量
```

### 4.3 信息价值(VPI)
```
VPI(E_j) = (∑ P(E_j=e_j) × EU(α_{e_j}|E_j=e_j)) - EU(α)

性质:
- VPI(E_j) ≥ 0 (非负性)
- VPI(E_j, E_k) ≠ VPI(E_j) + VPI(E_k) (不可加)
- VPI(E_j, E_k) = VPI(E_j) + VPI(E_k|E_j) (次序独立)
```

## 五、重要案例与应用

### 5.1 图书推荐系统
| 组件 | 说明 |
|-----|------|
| 类型 | Customer, Book |
| 属性 | Honest(c), Kindness(c), Quality(b) |
| 关系 | Recommendation(c,b) |
| 关系不确定性 | Author(b)未知时影响推荐网络结构 |

### 5.2 TrueSkill等级分系统
```
Skill(i) ~ N(μ, σ²)
Performance(i,g) ~ N(Skill(i), β²)
Win(i,j,g) if Game(g,i,j) then (Performance(i,g) > Performance(j,g))
```

### 5.3 引文匹配
- **未知对象**: 论文、作者、引用字符串
- **观测**: 带有噪声的引文字符串
- **推断目标**: 识别哪些引用指向同一论文

### 5.4 地震监测(NET-VISA)
- 检测失败率: 联合国系统 27.4% → NET-VISA 11.1%
- 多发现 50% 真实事件

### 6.1 渐进学习路径
```
阶段1: 理解RPM
    ↓ 掌握数据库语义
阶段2: 理解OUPM的数字语句和起源函数
    ↓ 理解对象作为生成历史

    ↓ 执行轨迹作为可能世界
阶段4: 综合应用
```

### 6.2 关键思考问题
1. RPM与OUPM的根本区别是什么？
2. 为什么OUPM需要起源函数？
3. 概率编程与传统概率建模相比有什么优势？
4. 多目标跟踪中的数据关联为什么困难？

### 6.3 常见误区
- ❌ RPM可以处理未知数量的对象
- ✅ RPM要求对象数量固定且已知
- ❌ OUPM落地后可以用标准贝叶斯网络算法求解
- ✅ OUPM世界空间可能无限，需要MCMC等近似推断

## 八、小结

概率编程代表了概率建模与编程语言的融合，其核心贡献包括：

1. **RPM**: 用一阶模板紧凑表示关系概率模型
2. **OUPM**: 处理存在不确定性和身份不确定性的开宇宙建模
3. **PPL**: 利用通用编程语言的表达能力定义复杂概率模型
4. **统一推断**: 通用MCMC等算法适用于各类概率程序

这一章将概率推理从固定的、有限的变量集合推广到了可以表示任意复杂、开放域问题的通用框架。
