### 题目 1：RPM基础建模 ⭐

**难度**: ⭐  
**涉及知识点**: RPM语法、依赖声明、基本随机变量

**题目**:  
考虑一个简单的学校评分系统。有2位老师T₁、T₂和3门课程C₁、C₂、C₃。每位老师有一个"严格度"(Strictness)属性，取值{1,2,3,4,5}；每门课程有一个"难度"(Difficulty)属性，取值相同。老师教授的每门课程有一个"评分"(Grade)，取值{A,B,C,D,F}。

(a) 定义这个域的RPM，包括类型签名和依赖声明。

(b) 如果使用贝叶斯网络直接建模，需要多少个节点？使用RPM需要指定多少个参数？

#### 逐步提示

<details>
<summary>点击查看提示</summary>

1. 类型有哪些？常量有哪些？
2. Strictness依赖于什么？Difficulty依赖于什么？
3. Grade应该依赖于哪些因素？
4. 考虑每位老师可能教授多门课程的情况
</details>

#### 详细解答

**(a) RPM定义**

```
类型: Teacher, Course
常量: T₁, T₂ : Teacher; C₁, C₂, C₃ : Course

函数签名:
  Strictness: Teacher → {1,2,3,4,5}
  Difficulty: Course → {1,2,3,4,5}
  Teaches: Teacher × Course → {true, false}
  Grade: Teacher × Course → {A,B,C,D,F}

依赖声明:
  Strictness(t) ~ <0.1, 0.2, 0.4, 0.2, 0.1>
  Difficulty(c) ~ <0.1, 0.3, 0.4, 0.15, 0.05>
  
  Grade(t,c) ~ if Teaches(t,c) then
                 GradeCPT(Strictness(t), Difficulty(c))
               else null
```

**(b) 参数对比**

| 建模方式 | 节点/变量数 | 参数数 |
|---------|-----------|-------|
| 贝叶斯网络 | Strictness: 2个<br>Difficulty: 3个<br>Grade: 6个(假设每对组合)<br>**总计: 11个节点** | GradeCPT: 2×5×5×5=250个概率值<br>加上先验: 2×5+3×5=25<br>**总计: 275个参数** |
| RPM | 同上(落地后) | 同上落地后<br>但模型描述只有:<br>2个先验+1个CPT定义<br>**约15个参数在模型描述中** |

**关键洞察**: RPM的优势在于通过模板共享，模型描述可以非常紧凑。即使扩展到100位老师和500门课程，模型描述的大小几乎不变。

### 题目 2：条件独立性分析 ⭐⭐

**难度**: ⭐⭐  
**涉及知识点**: 关系不确定性、上下文特定独立性

**题目**:  
考虑第15章的图书推荐RPM。假设我们观察到:
- Customer A对Book 1给出了5分推荐
- Customer A对Book 2也给出了5分推荐  
- Customer B对Book 1给出了5分推荐

(a) 在不知道任何顾客的Honest属性时，这些观测是否相互独立？

(b) 如果我们得知Customer A是诚实的(Honest(A)=true)，这对其他变量间的依赖关系有什么影响？

(c) 解释什么是"关系不确定性"，并说明它如何影响推断。

#### 逐步提示

<details>
<summary>点击查看提示</summary>

1. 如果Honest(A)未知，Recommendation(A,B1)和Recommendation(A,B2)都与Honest(A)相关
2. 如果Honest(A)已知，这两个推荐之间是否还有直接联系？
3. 关系不确定性涉及函数值未知导致网络结构变化
</details>

#### 详细解答

**(a) 条件独立性分析**

在没有证据时，三个推荐都通过Honest(A)和Honest(B)相互依赖：

```
Rec(A,B1) ← Honest(A) → Rec(A,B2)
     ↓
  Honest(B)? 
     ↓
Rec(B,B1)
```

因此它们**不是相互独立的**。

**(b) 给定Honest(A)=true的影响**

当Honest(A)已知：
- Rec(A,B1)和Rec(A,B2)变得条件独立(给定Honest(A)和各自的Quality)
- 但它们都与Quality(B1)相关
- Rec(A,B1)和Rec(B,B1)都提供关于Quality(B1)的证据

条件独立性图变为：
```
Rec(A,B1) ← Quality(B1) → Rec(B,B1)
                ↑
           Quality(B2) → Rec(A,B2)
```

**(c) 关系不确定性**

**定义**: 当函数值(如Author(b))不确定时，依赖结构本身变得不确定。

**例子**: 如果Author(B2)可能是A1或A2，那么Fan(C1,A1)和Fan(C1,A2)都可能是Rec(C1,B2)的父节点。MCMC采样时，每个状态指定一个具体的Author(B2)值，从而确定网络结构。

### 题目 3：OUPM数字语句 ⭐⭐

**难度**: ⭐⭐  
**涉及知识点**: 数字语句、泊松分布、起源函数

**题目**:  
考虑一个蚂蚁巢穴模型，其中有以下数字语句：

```
#Nest ~ Poisson(5)
#Ant(Nest=n) ~ Poisson(100)
```

(a) 解释这个模型的语义。一个可能世界中最多可能有多少只蚂蚁？

(b) 如果改为使用数量级分布OM(3,1)表示#Ant，解释其含义。

(c) 写出在这个模型中，第2个巢穴的第50只蚂蚁的完整对象表示。

#### 逐步提示

<details>
<summary>点击查看提示</summary>

1. Poisson分布有无限支撑，理论上可以有任意多个巢穴
2. OM(3,1)表示以10³为中心，标准差为一个数量级
3. 对象表示需要包含完整的生成历史
</details>

#### 详细解答

**(a) 模型语义**

- 巢穴数量服从均值为5的泊松分布
- 每个巢穴中的蚂蚁数量服从均值为100的泊松分布

**最大蚂蚁数**: 理论上无限(泊松分布有无限支撑)，但概率呈指数衰减。

期望蚂蚁数: E[#Ant] = E[E[#Ant|#Nest]] = 5 × 100 = 500

**(b) OM(3,1)含义**

**OM(μ,σ)**表示数量的对数(以10为底)服从均值为μ、标准差为σ的正态分布。

OM(3,1):
- 均值: 10³ = 1000只蚂蚁
- 标准差: 一个数量级(即大多数巢穴有10²到10⁴只蚂蚁)

**(c) 对象表示**

```
⟨Ant, ⟨Nest, ⟨Nest, _, 2⟩⟩, 50⟩
```

解释:
- 类型: Ant
- 起源: 属于第2个巢穴
- 索引: 该巢穴中的第50只蚂蚁

### 题目 4：概率程序语义 ⭐⭐

**难度**: ⭐⭐  
**涉及知识点**: 生成程序、执行轨迹、轨迹概率

**题目**:  
考虑以下简单的概率程序：

```
function FLIP-COIN():
    n ~ UniformChoice({1, 2, 3})
    heads = 0
    for i = 1 to n:
        outcome ~ Bernoulli(0.5)
        if outcome = 1 then heads = heads + 1
    return (n, heads)
```

(a) 列出所有可能的执行轨迹及其概率。

(b) 计算返回(2, 1)的总概率。

(c) 如果观察到返回(2, 1)，解释MCMC推断会如何探索其他可能的轨迹。

#### 逐步提示

<details>
<summary>点击查看提示</summary>

1. 轨迹包括所有随机选择: n的值和每次硬币投掷的结果
2. 轨迹概率是各随机选择概率的乘积
3. MCMC通过修改轨迹来探索，如改变n的值或某个outcome
</details>

#### 详细解答

**(a) 执行轨迹**

| 轨迹 | n | 投掷序列 | 结果 | 概率 |
|-----|---|---------|------|------|
| ω₁ | 1 | [H] | (1,1) | 1/3 × 1/2 = 1/6 |
| ω₂ | 1 | [T] | (1,0) | 1/3 × 1/2 = 1/6 |
| ω₃ | 2 | [H,H] | (2,2) | 1/3 × 1/4 = 1/12 |
| ω₄ | 2 | [H,T] | (2,1) | 1/3 × 1/4 = 1/12 |
| ω₅ | 2 | [T,H] | (2,1) | 1/3 × 1/4 = 1/12 |
| ω₆ | 2 | [T,T] | (2,0) | 1/3 × 1/4 = 1/12 |
| ω₇-ω₁₄ | 3 | [H,H,H]到[T,T,T] | (3,0)到(3,3) | 各1/24 |

**(b) P(return=(2,1))**

有两条轨迹返回(2,1): ω₄和ω₅

P = 1/12 + 1/12 = 1/6

**(c) MCMC推断**

给定证据return=(2,1)，MCMC会：

1. **初始化**: 从先验采样，如ω₄: n=2, outcomes=[H,T]

2. **提议移动**:
   - 修改n: 提议n=3，但需要调整outcomes以保持与证据一致
   - 修改outcomes: 提议[T,H]而不是[H,T]

3. **接受/拒绝**: 根据提议概率比率决定是否接受新轨迹

4. **混合**: 经过多次迭代，采样频率逼近后验分布

### 题目 5：多目标跟踪数据关联 ⭐⭐⭐

**难度**: ⭐⭐⭐  
**涉及知识点**: 数据关联、身份不确定性、MCMC

**题目**:  
考虑2个时间步的简化多目标跟踪问题：
- 2个已知飞行器A₁, A₂
- t=1时观测到2个光点B₁, B₂
- t=2时观测到2个光点B₃, B₄

假设每个光点恰好由一个飞行器产生。

(a) 有多少种可能的关联假设(将光点分配给飞行器)？

(b) 如果引入虚假警报的可能性(某些光点可能不对应任何飞行器)，如何修改OUPM？

(c) 解释为什么MCMC适合解决这类问题。

#### 逐步提示

<details>
<summary>点击查看提示</summary>

1. 每个时间步，2个光点分配给2个飞行器: 2! = 2种方式
2. 两个时间步独立(不考虑平滑)，总共2×2=4种
3. 如果考虑平滑(当前关联影响过去)，空间更大
</details>

#### 详细解答

**(a) 关联假设数量**

每个时间步，将2个光点分配给2个飞行器：2! = 2种方式

两个时间步: 2 × 2 = 4种基本关联假设

但如果考虑平滑(如MCMC所做的)，可以改变对过去的关联决策，所以实际搜索空间更大。

对于T个时间步、n个目标：(n!)ᵀ 种假设

**(b) 引入虚假警报的OUPM修改**

```
#Blip(Time=t) ~ Poisson(λ_f)  // 虚假警报光点

#Blip(Source=a, Time=t) ~ if InFlight(a,t) 
                           then Bernoulli(DetectionProb(X(a,t)))
                           else 0
                           
Z(b) ~ if Source(b) = null 
       then UniformZ(R)      // 虚假警报: 空间均匀分布
       else N(H·X(Source(b), Time(b)), Σ_z)
```

**(c) MCMC的优势**

1. **避免指数级展开**: 不需要枚举所有(n!)ᵀ种假设

2. **局部更新**: 每次迭代只改变一个或几个时间步的关联

3. **平滑能力**: 可以在获得新证据后重新评估过去的关联

4. **Rao-Blackwellization**: 给定关联假设，用卡尔曼滤波精确计算状态

### 题目 6：引文匹配建模 ⭐⭐⭐⭐

**难度**: ⭐⭐⭐⭐  
**涉及知识点**: 开宇宙模型、身份不确定性、生成模型

**题目**:  
设计一个OUPM模型来解决引文匹配问题。考虑以下场景：
- 研究人员撰写论文
- 论文被其他论文引用
- 引文字符串可能有拼写错误、缩写等

(a) 定义模型的类型、起源函数和数字语句。

(b) 描述如何从该模型生成引文字符串。

(c) 给定一组引文字符串作为证据，推断的目标是什么？

#### 逐步提示

<details>
<summary>点击查看提示</summary>

1. 类型包括: Researcher, Paper, Citation
2. 需要考虑论文-作者关系、引用关系
3. 引文字符串生成应考虑文本错误模型
</details>

#### 详细解答

**(a) OUPM模型定义**

```
类型: Researcher, Paper, Citation

数字语句:
  #Researcher ~ OM(3, 1)           // 约1000研究人员
  #Paper(Author=r) ~ if Professor(r) 
                     then OM(1.5, 0.5) 
                     else OM(1, 0.5)

起源函数:
  origin Researcher Author(Paper)

随机变量:
  Name(r) ~ NamePrior()
  Title(p) ~ PaperTitlePrior()
  PubCited(c) ~ UniformChoice({Paper p})
  Text(c) ~ HMMGrammar(
              Name(Author(PubCited(c))),
              Title(PubCited(c))
            )
```

**(b) 生成过程**

1. 采样#Researcher，为每个r生成Name(r)和Professor(r)
2. 为每位作者采样#Paper，生成每篇论文的Title(p)
3. 对每个引用c，采样PubCited(c)(指向被引论文)
4. 根据被引论文的作者名和标题，用HMM生成带噪声的Text(c)

**(c) 推断目标**

给定证据: Text(c₁)="Lashkari et al 94", Text(c₂)="Metral M. Lashkari..."

推断目标:
- PubCited(c₁) = PubCited(c₂)? (两个引用是否指向同一论文?)
- 如果是，被引论文的作者和标题是什么?
- 每个引用字符串的作者名变体对应哪个Researcher?

### 题目 7：等级分系统推断 ⭐⭐⭐⭐

**难度**: ⭐⭐⭐⭐  
**涉及知识点**: RPM推断、贝叶斯等级分、消息传递

**题目**:  
考虑第15章描述的贝叶斯等级分系统：

```
Skill(i) ~ N(μ, σ²)
Performance(i,g) ~ N(Skill(i), β²)
Win(i,j,g) if Game(g,i,j) then (Performance(i,g) > Performance(j,g))
```

(a) 解释为什么这个模型可以处理团队游戏。

(b) 给定以下比赛结果，推断玩家技能的后验分布：
- 游戏1: A vs B, A胜
- 游戏2: B vs C, B胜  
- 游戏3: A vs C, C胜

(c) 讨论精确推断的复杂性和近似方法。

#### 逐步提示

<details>
<summary>点击查看提示</summary>

1. 团队游戏可通过Performance的聚合(如求和)来建模
2. 比赛结果提供了Skill之间的相对比较信息
3. Win变量是非高斯的(阈值函数)，使精确推断困难
</details>

#### 详细解答

**(a) 团队游戏处理**

对于团队t在游戏g中的表现：
```
TeamPerformance(t,g) = Σ_{i∈t} Performance(i,g)
Win(t₁,t₂,g) = TeamPerformance(t₁,g) > TeamPerformance(t₂,g)
```

即使不知道个人表现，只要团队构成在不同游戏间变化，就可以估计个人技能。

**(b) 后验推断**

**证据**: Win(A,B,g₁)=true, Win(B,C,g₂)=true, Win(C,A,g₃)=true

**循环关系**: A>B>C>A，形成了非传递性证据

**推断结果**:
- 由于A>B且B>C，预期A>C
- 但观察到C>A，这表明要么A和C技能接近(噪声导致)，要么证据有矛盾

后验分布会显示A、B、C的技能高度重叠，方差较大。

**(c) 推断复杂性**

**复杂性**: Win变量涉及不等式，导致后验非高斯。

**近似方法**:
1. **消息传递**: 期望传播(EP)，将非高斯消息近似为高斯
2. **MCMC**: Gibbs采样Skill变量
3. **变分推断**: 用高斯分布近似后验

TrueSkill系统使用EP，每天服务数亿用户。

### 题目 8：提升推断 ⭐⭐⭐⭐

**难度**: ⭐⭐⭐⭐  
**涉及知识点**: 落地、提升推断、对称性利用

**题目**:  
考虑一个RPM，描述社交网络中的友谊关系：

```
类型: Person
常量: P₁, P₂, ..., Pₙ : Person

Popular(p) ~ Bernoulli(0.2)
Friendly(p) ~ if Popular(p) then N(0.8, 0.1) else N(0.3, 0.1)

Friends(p₁, p₂) ~ Bernoulli(sigmoid(Friendly(p₁) + Friendly(p₂) - 1))
```

(a) 解释为什么直接落地(生成完整贝叶斯网络)对于大n是不切实际的。

(b) 描述提升推断如何利用对称性避免完全落地。

(c) 在什么情况下提升推断会失效？

#### 逐步提示

<details>
<summary>点击查看提示</summary>

1. Friends谓词有O(n²)个实例
2. 许多人可能具有相同的属性值组合
3. 如果证据打破对称性，提升可能失效
</details>

#### 详细解答

**(a) 落地问题**

落地网络包含:
- n个Popular和Friendly节点
- C(n,2) = O(n²)个Friends节点

对于n=10⁶用户，Friends节点约5×10¹¹个，不可行。

**(b) 提升推断**

**核心思想**: 将对称的落地因子分组处理

```
1. 识别等价类: 
   - 所有非证据Person节点按属性值分组
   
2. 聚合计算:
   - 不是计算每个(p₁,p₂)对的Friends概率
   - 而是计算"类型A和类型B之间有多少对朋友"
   
3. 计数因子:
   - 使用组合计数代替枚举
```

**复杂度降低**: 从O(n²)到O(k²)，k是等价类数量

**(c) 提升失效情况**

1. **证据打破对称性**: 如果我们知道Friends(P₁,P₂)=true，P₁和P₂就不再可交换

2. **复杂约束**: 涉及多个对象的全局约束

3. **连续变量**: Friendly是连续的，等价类可能是无限的

### 题目 9：复杂世界追踪的POMDP建模 ⭐⭐⭐⭐⭐

**难度**: ⭐⭐⭐⭐⭐  
**涉及知识点**: OUPM、POMDP、数据关联、在线推断

**题目**:  
设计一个结合OUPM和POMDP的框架来处理自动驾驶场景中的多目标跟踪：

**场景**: 自动驾驶汽车需要跟踪周围的车辆、行人和 cyclist。对象可能进入/离开视野，传感器有噪声。

(a) 定义模型的状态空间、观测和动作。

(b) 解释如何结合OUPM的对象存在不确定性和POMDP的部分可观测性。

(c) 讨论实时性要求下的近似推断策略。

#### 详细解答

**(a) 状态、观测和动作**

```
状态空间(隐变量):
  - 自车状态: 位置、速度、朝向
  - 动态对象: #Vehicle, #Pedestrian, #Cyclist
    - 对每个对象: 位置、速度、类型、意图
  - 静态环境: 道路结构

观测(带噪声):
  - 激光雷达点云
  - 摄像头图像
  - 雷达检测
  
动作:
  - 纵向: 加速、减速、保持
  - 横向: 左转、右转、保持
```

**(b) OUPM-POMDP结合**

```
┌────────────────────────────────────────────────────────────┐
│                   分层表示                                  │
├────────────────────────────────────────────────────────────┤
│  OUPM层 (对象层)                                            │
│    #Vehicle ~ Poisson(λ_v)  // 进入视野的速率              │
│    Vehicle(i)状态: (x, y, vx, vy, type)                    │
│                                                             │
│  转移:                                                      │
│    X_t(i) ~ N(F·X_{t-1}(i), Σ_x)  // 运动模型              │
│    #Vehicle_t ~ ...  // 新对象以一定概率出现               │
│                                                             │
│  POMDP层 (决策层)                                           │
│    信念状态: b_t = P(所有对象状态 | 历史观测)              │
│    动作选择: 基于当前信念的期望效用最大化                  │
│    观测更新: 将传感器读数与对象关联(数据关联)              │
└────────────────────────────────────────────────────────────┘
```

**关键整合点**:
- OUPM处理"有多少对象"和"它们是谁"
- POMDP处理"给定当前不确定性，我应该做什么"

**(c) 实时近似策略**

1. **粒子滤波+MCMC**:
   - 用粒子表示对象配置
   - 每个粒子内部用卡尔曼滤波跟踪具体对象
   - 粒子间的MCMC移动处理数据关联

2. **分离时间尺度**:
   - 高频: 对象跟踪(100Hz)
   - 低频: 高层决策(10Hz)

3. **注意力机制**:
   - 只维护附近对象的精确信念
   - 远处对象用简化模型

### 题目 10：概率程序中的推断编译 ⭐⭐⭐⭐⭐

**难度**: ⭐⭐⭐⭐⭐  
**涉及知识点**: 概率编程、推断编译、神经网络

**题目**:  
推断编译是将概率程序自动转换为高效推断代码的技术。

(a) 解释为什么通用MCMC在复杂概率程序上可能效率低下。

(b) 描述如何使用神经网络学习自适应提议分布。

(c) 设计一个针对特定领域(如OCR)的混合推断策略。

#### 详细解答

**(a) MCMC效率问题**

```
┌────────────────────────────────────────────────────────────┐
│                  MCMC在PPL中的挑战                          │
├────────────────────────────────────────────────────────────┤
│  1. 高维空间: 程序轨迹包含所有随机选择                      │
│     - 如256×128图像有32768个像素变量                      │
│                                                             │
│  2. 混合困难: 改变早期选择(如"字母数量")需要               │
│     级联修改后续所有变量                                     │
│                                                             │
│  3. 后验复杂: 多峰、高度相关、约束复杂                      │
│                                                             │
│  4. 拒绝率高: 随机提议很少符合观测约束                      │
└────────────────────────────────────────────────────────────┘
```

**(b) 神经网络自适应提议**

```
训练阶段:
  1. 从先验生成合成数据: (观测, 隐藏变量)对
  2. 训练神经网络 q_φ(隐藏|观测) 近似后验
  3. 网络输出提议分布的参数

推断阶段:
  1. 给定观测, 网络输出提议分布
  2. 使用提议分布进行重要性采样或MCMC
  3. 可选: 在线微调

优势:
  - 数据驱动的提议, 拒绝率低
  - 摊销推断(amortized inference): 训练一次, 多次使用
```

**(c) OCR混合推断策略**

**1. RPM与标准贝叶斯网络的主要区别是什么？**

A. RPM使用连续变量，贝叶斯网络使用离散变量  
B. RPM通过模板定义，可自动展开为不同规模的网络  
C. RPM只能表示因果关系，贝叶斯网络表示相关性  
D. RPM不支持条件概率表  

<details>
<summary>答案</summary>

**答案: B**

RPM使用一阶模板(带参数的依赖声明)，对于给定常量集合可以自动展开为贝叶斯网络。这使得相同模型描述可用于不同规模的问题。

</details>

**2. 在OUPM中，"数字语句"(Number Statement)的主要作用是什么？**

A. 指定随机变量的取值范围  
B. 定义对象数量的概率分布  
C. 计算条件概率  
D. 确定观测变量的精度  

<details>
<summary>答案</summary>

**答案: B**

数字语句如`#Customer ~ UniformInt(1, 3)`定义了该类型对象数量的概率分布，允许世界中对象数量不确定。

</details>

**3. 以下哪种情况最适合使用OUPM而不是RPM？**

C. 确定性规划问题  
D. 完全可观测的棋盘游戏  

<details>
<summary>答案</summary>

**答案: B**

OUPM适用于存在不确定性(不知道有多少对象)的场景。命名实体识别中，实体数量事先未知，是典型的OUPM应用。

</details>

**4. 在概率程序中，"执行轨迹"是指什么？**

执行轨迹是程序运行时所有随机选择(如`x ~ Distribution`)的具体取值序列，对应概率模型中的一个可能世界。

**6. MCMC在OUPM中的优势是什么？**

A. MCMC可以提供精确解  
B. MCMC不需要指定先验  
C. MCMC通过采样完整世界来处理关系不确定性  
D. MCMC总是比变分推断快  

<details>
<summary>答案</summary>

**答案: C**

MCMC通过采样完整的可能世界(包括所有关系和函数值)来探索空间。在每个MCMC状态中，关系结构完全已知，便于概率计算。

</details>

**7. 以下哪个是"身份不确定性"的例子？**

<details>
<summary>答案</summary>

**答案: B**

身份不确定性是关于"两个观测是否指向同一对象"的不确定性，如引文匹配中判断两个字符串是否引用同一论文。

</details>

**8. 提升推断(Lifted Inference)的主要思想是？**

A. 使用深度学习加速推断  
B. 利用对称性避免完全落地  
C. 将推断提升到云端计算  
D. 使用高阶逻辑表示  

<details>
<summary>答案</summary>

**答案: B**

提升推断识别模型中的对称性(可交换的对象)，通过聚合计算代替逐个处理，避免生成完整的落地网络。

</details>

**9. 在TrueSkill等级分系统中，团队游戏的处理方式是什么？**

D. 只考虑表现最好的团队成员

**答案: B**

TrueSkill近似假设团队表现是成员个人表现之和，这样即使不知道个人贡献，也能从团队结果推断个人技能。

</details>

**10. POMDP与OUPM结合的主要挑战是什么？**

A. POMDP不能处理连续状态

C. OUPM不能表示奖励函数  
D. 两者都假设完全可观测性  

<details>
<summary>答案</summary>

**答案: B**

POMDP的信念状态是状态上的概率分布(连续空间)，结合OUPM后还需要考虑对象数量不确定性，使信念空间更加复杂。

</details>

## 考点解析

### 核心概念对照表

| 考点 | 关键理解 | 常见错误 |
|-----|---------|---------|
| RPM | 模板化BN，固定对象 | 认为可处理变数量对象 |
| OUPM | 数字语句+起源函数 | 混淆可能世界与执行轨迹 |
| 存在不确定性 | 对象数量不确定 | 与身份不确定性混淆 |
| 身份不确定性 | 观测-对象对应不确定 | 与属性值不确定性混淆 |
| 概率程序 | 轨迹=世界 | 忽视随机性的条件性质 |
| 数据关联 | 多目标跟踪核心 | 与目标检测混淆 |
| 提升推断 | 利用对称性 | 认为总是适用 |

### 难度分布总结

- **基础题(⭐-⭐⭐)**: RPM/OUPM语法、基本概念
- **综合题(⭐⭐⭐-⭐⭐⭐⭐)**: 建模应用、推断算法
- **挑战题(⭐⭐⭐⭐⭐)**: 前沿问题、系统设计
