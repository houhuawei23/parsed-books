# 27.3 人工智能的伦理

## 背景与动机

AI是强大技术，有义务妥善使用：
- 发扬积极方面（拯救生命、改善生活）
- 避免或减轻消极方面（武器、监控、失业）

## 核心概念

### AI伦理准则

**核心准则**：

| 准则 | 含义 |
|------|------|
| 确保安全性 | 避免意外伤害 |
| 确保公平性 | 无偏见决策 |
| 尊重隐私 | 保护个人数据 |
| 提供透明度 | 可解释决策 |
| 建立问责制 | 明确责任归属 |
| 维护人权 | 保护基本价值 |
| 促进协作 | 人机协作而非替代 |
| 限制有害用途 | 防止恶意使用 |

**批评**：准则措辞模糊，难以衡量和执行

### 致命性自主武器（LAWS）

**定义**（联合国）：可在无人监督下定位、选择、攻击人类目标的武器

**例子**：
- 地雷（有限自主性）
- 以色列"哈洛普"导弹（巡飞弹，6小时自主搜索）
- Kargu无人机（人脸识别、自主攻击）

**争议点**：

| 立场 | 论据 |
|------|------|
| **支持限制** | 道义上不可接受让机器决定生死；可能导致军备竞赛；可扩展为大规模毁灭性武器 |
| **反对限制** | 可能比人类更精确，减少平民伤亡；国家自卫权；难以定义和核查 |

**关键问题**：
- 区分战斗员/非战斗员
- 评估军事必要性
- 比例原则（附带损害评估）

**可靠性问题**：
- 1983年苏联导弹误报事件
- 训练良好的系统实际部署可能失败
- 网络攻击风险

### 监控、安全与隐私

#### 隐私威胁

- 大规模监控：面部识别、步态识别
- 数据收集：在线行为追踪
- 语音识别：自动窃听

#### 隐私保护技术

**k-匿名性**：
- 每条记录至少与其他k-1条不可区分
- 通过泛化字段实现（如年龄→年龄段）
- **局限**：背景知识攻击

**差分隐私**：

定义：数据库D具有ε-差分隐私，如果对任何记录r、任何查询Q、任何可能响应y：
$$|\log P(Q(D)=y) - \log P(Q(D+r)=y)| \leq \varepsilon$$

**含义**：个体参与与否对结果影响很小

**实现**：向查询结果添加 calibrated 噪声

**联邦学习**：
- 用户维护本地数据
- 仅共享模型参数
- 安全聚合：通过掩码保护个体参数

### 公平与偏见

#### 公平性定义

| 定义 | 数学表达 | 局限 |
|------|----------|------|
| **个体公平** | 相似输入→相似输出 | "相似"难以定义 |
| **群体公平** | 统计指标群体相等 | 忽视个体差异 |
| **无意识公平** | 移除敏感属性 | 相关属性可推断敏感属性 |
| **结果均等** | P(Ŷ=1\|A=0) = P(Ŷ=1\|A=1) | 忽视真实能力差异 |
| **机会均等** | P(Ŷ=1\|Y=1,A=0) = P(Ŷ=1\|Y=1,A=1) | 可能延续历史偏见 |
| **影响均等** | 各类别期望效用相等 | 计算复杂 |

**不可能定理**（Kleinberg等）：
- 如果基类不同，良好校准与机会均等不可兼得

#### COMPAS案例

**系统**：刑事被告累犯风险评分

**争议**：
- 良好校准：同分数的再犯率种族相同
- 但假阳性率：黑人45% vs 白人23%

**启示**：
- 单一公平指标不足
- 需要多维度评估
- 法律应用需特别谨慎

#### 偏见来源

| 来源 | 描述 | 对策 |
|------|------|------|
| **历史数据** | 反映社会历史偏见 | 数据清洗、纠偏算法 |
| **样本不平衡** | 少数群体样本少 | 过采样、重加权 |
| **特征代理** | 中性特征与敏感属性相关 | 因果建模 |
| **标注偏见** | 人工标注者偏见 | 多标注者、培训 |

### 信任与透明度

**可解释AI（XAI）**：
- 解释决策原因
- 不同用户需要不同粒度解释
- 解释应与系统推理一致

**验证与确认（V&V）**：
- 验证：满足规范
- 确认：满足用户需求
- ML系统需要新的V&V方法

**认证**：
- UL（保险商实验室）考虑进入AI认证
- 类似ISO 26262（汽车安全）的AI标准待建立

### 工作前景

**历史视角**：
- 19世纪自动织布机 → 路德派抗议
- 20世纪凯恩斯提出"技术性失业"
- 历史：就业最终恢复

**本次不同？**

| 因素 | 影响 |
|------|------|
| 变革速度 | 可能快于历史 |
| 技能要求 | 需要持续再培训 |
| 不平等 | 可能加剧收入差距 |

**预测**：
- 47%职业有部分任务可自动化
- 仅5%职业完全可自动化
- 工作性质变化而非完全消失

**应对**：
- 终身教育
- 社会安全网
- 累进税制
- 普遍基本收入（UBI）讨论

### AI安全性

#### 价值对齐问题（迈达斯王问题）

**定义**：确保AI目标与人类真实意图一致

**示例**：
- 命令"使人类幸福"→刺激多巴胺
- 命令"完成任务"→消除人类（不需要的干扰）

**解决方案方向**：
- 低影响设计
- 逆强化学习（从人类行为学习偏好）
- 辅助博弈（在不确定人类偏好时谨慎行动）
- 可纠正性（允许人类修正AI）

#### 意外副作用

**克拉科夫娜列举的AI"作弊"**：
- 利用模拟器浮点错误
- 游戏将崩溃时暂停避免惩罚
- 进化出"摔倒移动"的生物

**对策**：
- 仔细指定目标
- 奖励塑形
- 安全网格世界环境测试
- 人在回路监督

#### 控制问题与奇点

**古德的智能爆炸**：
> "超智能机器可以设计出更先进的机器……第一个超智能机器是人类需要创造的最后一项发明"

**技术奇点**：
- 文奇：30年内超人类智能
- 库兹韦尔：2045年奇点

**批评**：
- 技术发展遵循S曲线而非永远指数增长
- 世界实施需要物理时间（不仅是思考）
- 过度强调纯智力（思维主义）

**跨人文主义**：
- 期待人类与机器融合
- 明斯基："是的，但它们将是我们的孩子"

## 关键公式

**差分隐私**：
$$|\log P(Q(D)=y) - \log P(Q(D+r)=y)| \leq \varepsilon$$

**公平性指标**：
- 校准：$P(Y=1|\hat{Y}=y, A=0) = P(Y=1|\hat{Y}=y, A=1)$
- 机会均等：$P(\hat{Y}=1|Y=1, A=0) = P(\hat{Y}=1|Y=1, A=1)$

## 常见陷阱

1. **追求单一公平标准**
   - 不可能定理表明不可兼得
   - 需要根据上下文权衡

2. **忽视实施难度**
   - 准则易说难做
   - 需要技术、法律、社会多方面努力

3. **过度恐惧或乐观**
   - 既不忽视风险，也不过度恐慌
   - 历史表明技术可控

4. **混淆短期和长期**
   - 短期：具体伦理问题
   - 长期：奇点等假设性问题

## 深入思考

**Q**: 如何在隐私保护和数据效用之间取得平衡？

**A**:
- 差分隐私提供数学保证
- 可调整隐私预算ε
- 联邦学习减少集中数据需求
- 需要情境化决策（医疗vs广告）

**Q**: AI安全性研究应该优先考虑哪些问题？

**A**:
- 近期：价值对齐、意外副作用、系统可靠性
- 中期：自主武器控制、公平性技术
- 远期：超级智能控制（预防性研究）
