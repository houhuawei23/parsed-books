### 23.1 语言模型基础

#### 练习 1: n元模型概率计算

给定以下二元概率（假设）：
- P(the|⟨S⟩) = 0.25
- P(cat|the) = 0.003
- P(sat|cat) = 0.001
- P(on|sat) = 0.2
- P(the|on) = 0.4
- P(mat|the) = 0.002
- P(⟨/S⟩|mat) = 0.3

**问题**：计算句子"the cat sat on the mat"的二元模型概率。

**解答**：

$$
\begin{aligned}
P(\text{the cat sat on the mat}) &= P(\text{the}|\langle S\rangle) \times P(\text{cat}|\text{the}) \times P(\text{sat}|\text{cat}) \\
&\quad \times P(\text{on}|\text{sat}) \times P(\text{the}|\text{on}) \times P(\text{mat}|\text{the}) \times P(\langle/S\rangle|\text{mat}) \\
&= 0.25 \times 0.003 \times 0.001 \times 0.2 \times 0.4 \times 0.002 \times 0.3 \\
&= 3.6 \times 10^{-12}
\end{aligned}
$$

#### 练习 2: 拉普拉斯平滑

给定训练语料库：
- 总词数 N = 10,000
- 词汇表大小 |V| = 1,000
- 单词"wumpus"出现次数 = 0

**问题**：
a) 计算"wumpus"的未平滑概率

b) 使用拉普拉斯平滑计算其平滑后概率

**解答**：

a) 未平滑概率：
$$P(\text{wumpus}) = \frac{0}{10000} = 0$$

b) 拉普拉斯平滑：
$$P_{Laplace}(\text{wumpus}) = \frac{0 + 1}{10000 + 1000} = \frac{1}{11000} \approx 9.09 \times 10^{-5}$$

#### 练习 3: 词性标注

给定HMM参数：
- 转移概率：P(VB|MD) = 0.8, P(NN|MD) = 0.1
- 发射概率：P(would|MD) = 0.3, P(think|VB) = 0.1

**问题**：句子"would think"，已知"would"是MD，求"think"最可能的词性。

**解答**：

使用维特比算法的简化版本：

$$
\begin{aligned}
P(VB|\text{would}, \text{think}) &\propto P(VB|MD) \times P(\text{think}|VB) \\
&= 0.8 \times 0.1 = 0.08
\end{aligned}
$$

$$
\begin{aligned}
P(NN|\text{would}, \text{think}) &\propto P(NN|MD) \times P(\text{think}|NN) \\
&= 0.1 \times P(\text{think}|NN)
\end{aligned}
$$

由于"think"作为名词很少见，$P(\text{think}|NN)$很小，因此VB更可能。

给定文法：
```

```
        S [1.0]
       / \
     NP   VP [0.7]
    / |   | \
 Det  N   V  NP [0.6]
  |   |   |  / \
 the cat chases Det N
                |   |
               the dog

概率计算：

$$
\begin{aligned}
P &= 1.0 \times 0.6 \times 1.0 \times 0.5 \times 0.7 \times 0.5 \times 0.6 \times 1.0 \times 0.5 \\
&= 1.0 \times 0.6 \times 0.5 \times 0.7 \times 0.5 \times 0.6 \times 0.5 \\
&= 0.0315
\end{aligned}
$$

### 练习 8: 平滑技术比较

**问题**：解释为什么拉普拉斯平滑在自然语言任务中效果不佳，并说明Kneser-Ney平滑的优势。

**解答**：

**拉普拉斯平滑的问题**：

1. **概率质量分配不均**：将所有未出现事件的概率均匀分配
2. **高估低频词**：对于大型词汇表，大量概率质量被浪费在\<UNK\>上
3. **相对频率失真**：改变了已观测事件的相对概率

**示例**：
- 词汇表|V|=50,000
- "the"出现1000次，"a"出现500次
- 拉普拉斯后：$P(the) = 1001/(N+50000)$，$P(a) = 501/(N+50000)$
- 相对频率从2:1变为约2:1（变化不大但累积效应大）

**Kneser-Ney平滑的优势**：

1. **绝对折扣**：从计数中减去固定值（而非加1）
2. **回退到更短历史**：利用低阶模型信息
3. ** continuation概率**：基于词汇的"延续能力"而非出现频率

$$
P_{KN}(w_i|w_{i-1}) = \frac{\max(c(w_{i-1}, w_i) - d, 0)}{c(w_{i-1})} + \lambda(w_{i-1}) P_{continuation}(w_i)
$$

### 练习 9: 扩展文法设计

**问题**：设计处理主谓一致的扩展文法规则。

**解答**：

```
// 句子规则：要求NP和VP的人称/数一致
S(v) → NP(Sbj, pn, n) VP(pn, v)

// 名词短语：携带格、人称数、语义信息
NP(c, pn, n) → Pronoun(c, pn, n)
NP(c, pn, n) → Det PN Noun(c, pn, n)

// 动词短语：继承人称数
VP(pn, v) → Verb(pn, v) NP(Obj, pn, n)

// 词汇规则示例
Pronoun(Sbj, 1S, I) → I [0.1]
Pronoun(Obj, 1S, me) → me [0.05]
Pronoun(Sbj, 3S, he) → he [0.08]

Verb(1S, see) → see [0.02]
Verb(3S, see) → sees [0.02]

// 一致性情义
约束: NP(Sbj, pn1, _) + VP(pn2, _) 要求 pn1 = pn2
```

**有效句子**：
- "I see" (1S = 1S) ✓
- "He sees" (3S = 3S) ✓

**无效句子**：
- "I sees" (1S ≠ 3S) ✗
- "He see" (3S ≠ 1S) ✗

### 练习 10: 语义组合

**问题**：使用λ演算表示"Every student reads some book"的语义。

**解答**：

**分析**：存在两种量词作用域解读

**解读1（学生宽域）**：
$$
\forall s [Student(s) \Rightarrow \exists b [Book(b) \wedge Read(s, b)]]
$$

**解读2（书籍宽域）**：
$$
\exists b [Book(b) \wedge \forall s [Student(s) \Rightarrow Read(s, b)]]
$$

**λ表示**：

```
Det(∀) → λP λQ ∀x [P(x) ⇒ Q(x)]
Det(∃) → λP λQ ∃x [P(x) ∧ Q(x)]

Noun(student) → λx Student(x)
Noun(book) → λx Book(x)

Verb(read) → λy λx Read(x, y)
```

**组合推导**（解读1）：

```
NP1: Det(∀) Noun(student)
   = (λP λQ ∀x [P(x) ⇒ Q(x)])(λx Student(x))
   = λQ ∀x [Student(x) ⇒ Q(x)]

VP: Verb(read) NP2
   = (λy λx Read(x, y))(∃b [Book(b)])
   = λx Read(x, ∃b [Book(b)])

S: NP1 VP
   = (λQ ∀x [Student(x) ⇒ Q(x)])(λx Read(x, ∃b [Book(b)]))
   = ∀x [Student(x) ⇒ Read(x, ∃b [Book(b)])]
```

### 练习 12: 句法分析器复杂度分析

**问题**：证明CYK算法的时间复杂度为$O(n^3)$。

**解答**：

**CYK算法核心循环**：

```
for length = 2 to n:
    for i = 1 to n - length + 1:
        k = i + length - 1
        for j = i to k - 1:
            // 检查所有规则 X → Y Z
```

**复杂度分析**：

1. **外层循环**（length）：$n$ 次迭代
2. **中层循环**（i）：对每个长度，最多 $n$ 次迭代
3. **内层循环**（j）：对每个$(i,k)$，最多 $n$ 次分割点
4. **规则检查**：假设文法大小为常数

**总复杂度**：

$$
T(n) = \sum_{length=2}^{n} \sum_{i=1}^{n-length+1} \sum_{j=i}^{i+length-2} O(1)
$$

简化（最坏情况，假设所有循环到n）：

$$
T(n) = O(n) \times O(n) \times O(n) = O(n^3)
$$

**精确计算**：

子串数量：$\binom{n}{2} = O(n^2)$

对每个子串，尝试 $O(n)$ 个分割点

总计：$O(n^3)$

### 练习 13: 机器翻译对齐

**问题**：解释机器翻译中词汇对齐的挑战，并说明IBM Model 1的基本思想。

**解答**：

**对齐挑战**：

1. **一对多**：英语"not"可能对应汉语"不"、"没"、"别"等
2. **多对一**："New York"整体翻译为"纽约"
3. **空对齐**：英语冠词"the"在汉语中常省略
4. **词序差异**：日语动词在句末，英语在主语后

**IBM Model 1**：

**假设**：
- 词对齐独立（简化假设）
- 每个源词独立选择目标词

**模型**：

$$
P(f, a | e) = \prod_{j=1}^{m} t(f_j | e_{a_j})
$$

其中：
- $f$：目标语言句子
- $e$：源语言句子
- $a$：对齐
- $t$：翻译概率

**EM学习**：

E步：计算期望对齐
M步：更新翻译概率

迭代直至收敛。
