## 理解人类行为

### 人体姿态估计

**目标**：从图像/视频中估计人体关节的三维位置。

```
2D姿态：                    3D姿态：
┌─────────────┐            
│    ○        │              ○(头)
│   ╱│╲       │             ╱│╲
│  ╱ │ ╲      │            ○─○─○(肩)
│ ●──┼──●     │           ╱ │ ╲
│    │        │          ●  │  ●(肘)
│   ╱ ╲       │         ╱   │   ╲
│  ●   ●      │        ●    │    ●(手)
│ (膝) (膝)   │       (腿)  ●  (腿)
└─────────────┘            (髋)
```

**应用**：
- 游戏交互（Xbox Kinect）
- 运动分析
- 健康监测
- 虚拟现实

**挑战**：
- 遮挡（自遮挡、外部遮挡）
- 多人场景
- 服装变化

### 动作识别

**时间尺度问题**：

```
单帧图像：        短视频：          长视频：
"打开冰箱"       "从冰箱拿牛奶"     "准备点心"
┌────┐          ┌────┐┌────┐      ┌────┐┌────┐┌────┐
│🚪  │          │🚪  ││🥛  │      │🚪  ││🥛  ││🍪  │
│ 🖐️ │          │ 🖐️││ 🖐️│      │ 🖐️││ 🖐️││ 🖐️│
└────┘          └────┘└────┘      └────┘└────┘└────┘
   ↓               ↓                  ↓
低层动作        中层动作           高层活动
```

**当前技术水平**：
- 结构化行为（芭蕾舞、体操）：效果良好
- 自然行为：仍具挑战性

**挑战**：
- 环境依赖性（"游泳"分类器可能是"泳池检测器"）
- 行为组合（准备点心 = 拿牛奶 + 拿饼干 + ...）
- 同时发生的行为（唱歌 + 准备点心）

## 图像与文字交互

### 图像标注（Tagging）

**任务**：为图像分配关键词标签。

```
输入图像：                    输出标签：
┌─────────────────┐          • 猫
│    ╭───╮        │          • 街道
│    │🐱 │  🚮    │          • 垃圾桶
│    ╰───╯   🐟   │          • 鱼骨
│                 │
└─────────────────┘

问题：标签无法描述"猫正在从垃圾桶里拽出鱼骨"
```

### 图像标题生成（Captioning）

**任务**：生成描述图像的自然语言句子。

```
架构：CNN + RNN/Transformer

图像 → [CNN特征提取] → 特征向量 → [RNN解码] → "一只猫在街上"
         (编码器)                      (解码器)
```

**示例结果**：

```
好结果：                    失败案例：
"一个小孩在吃食物"         "一只棕色小熊坐在草地上"
"一个年轻男孩在吃蛋糕"      （实际：松鼠）
```

**数据集**：COCO数据集（20万+图像，每图5个标题）

### 视觉问答（VQA）

**任务**：回答关于图像的自然语言问题。

```
Q: 猫戴了什么？
A: 帽子 ✓

Q: 天气怎么样？
A: 下雨 ✓

Q: 披萨上有几个洞？
A: 8 ✗ （系统不懂什么是"洞"）
```

**挑战**：
- 需要结合视觉理解和语言理解
- 计数、推理困难
- 容易受背景颜色干扰

### 应用：虚拟现实与影视制作

**将虚拟角色放入真实场景**：

```
步骤：
1. 估计图像深度图
2. 估计光照条件
3. 将3D模型嵌入场景
4. 使用物理渲染
5. 与原图融合

结果：虚拟物体与真实场景无缝融合
```

## 图像生成与转换

### 成对图像转换

**任务**：学习从X类型图像到Y类型图像的映射。

```
应用示例：

航空影像 → 道路图：
卫星图    →   地图
┌─────┐    ┌─────┐
│ 🏠🏢│    │ ▓▓▓ │
│ ═══ │ →  │═════│
│ 🌲🌳│    │ ··· │
└─────┘    └─────┘

模糊图像 → 清晰图像：
┌─────┐    ┌─────┐
│  ~~ │    │  📷 │
│ 🌫️🌫️│ →  │ 🏞️  │
│  ~~ │    │  📷 │
└─────┘    └─────┘
```

**训练**：使用成对数据集 + GAN损失

### 未配对图像转换（CycleGAN）

**任务**：两个域之间没有配对数据，只有各自的数据集。

```
马 → 斑马：

X域(马)              Y域(斑马)
┌─────┐              ┌─────┐
│ 🐴  │   G: X→Y    │ 🦓  │
└─────┘ ───────────→ └─────┘
   ↑                    ↓
   │                    │
   │   F: Y→X    ┌─────┐
   └──────────── │ 🐴' │
                 └─────┘

循环一致性：F(G(X)) ≈ X
```

### 风格迁移

**任务**：将内容图像的内容与艺术图像的风格结合。

```
内容：        风格：        结果：
┌─────┐      ┌─────┐      ┌─────┐
│ 🐱  │  +   │🎨抽象│  =  │🎨🐱🎨│
│照片 │      │画作 │      │艺术猫│
└─────┘      └─────┘      └─────┘
```

**方法**：
- 使用预训练CNN
- 早期层表示风格
- 晚期层表示内容
- 优化生成图像同时匹配两者

### 深度伪造（Deepfake）

**应用**：
- 电影制作（《侠盗一号》中的凯丽·费雪）
- 虚拟主播
- 隐私保护（生成医学影像数据集）

**风险**：
- 假新闻传播
- 身份欺诈

**对策**：
- 伪造检测算法
- 数字水印技术

### 自动驾驶

┌─────────────────────────────────────┐
│  车道检测        → 横向控制          │
│  前车距离        → 纵向控制          │

- 仅使用摄像头

- 输出：3D位置和速度

### 机器人导航

**经典方法**：

```
地图构建(SLAM) → 路径规划 → 运动控制
      ↑              ↓
      └────定位──────┘
```

**端到端学习**：

```

无需显式建图，直接学习从感知到动作的映射
```

## 常见陷阱

### 陷阱1：环境过拟合

```
训练：游泳池里的游泳者
测试：河流里的游泳者
结果：分类失败

原因：模型学习的是背景而非动作
```

### 陷阱2：罕见情况

```
训练集中没有：
- 骑独轮车的行人
- 倒立的汽车
- 戴头盔的猫

遇到这些情况时，系统行为不可预测
```

### 陷阱3：对抗攻击

```
微小扰动 → 完全错误的结果

停车标志 + 贴纸 → 系统识别为"限速60"
```

### 陷阱4：伦理问题

**隐私**：
- 无处不在的监控
- 人脸识别滥用

**偏见**：
- 对某些群体的识别准确率较低
- 性别/种族偏见

## 小结

1. **理解人类行为**：
   - 姿态估计已较成熟
   - 动作识别仍具挑战
   - 注意时间尺度和行为组合

2. **图像-文字交互**：
   - 标注：关键词分配
   - 标题：句子生成
   - VQA：回答问题
   - 都存在描述不准确的问题

3. **三维重建应用**：
   - 建筑施工管理
   - 虚拟现实
   - 自动驾驶高精地图

4. **图像生成**：
   - 成对/未配对转换
   - 风格迁移
   - 深度伪造（机遇与风险并存）

5. **视觉控制运动**：
   - 自动驾驶：多传感器融合
   - 机器人：SLAM或端到端学习

6. **注意事项**：
   - 避免环境过拟合
   - 考虑罕见情况
   - 注意伦理问题
