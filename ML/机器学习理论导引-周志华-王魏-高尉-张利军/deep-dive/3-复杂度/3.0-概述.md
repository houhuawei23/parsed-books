## 1. 背景与意义

从第2章介绍的PAC学习理论可以看出，PAC可学性与假设空间 $\mathcal{H}$ 的复杂程度密切相关。假设空间越复杂，从中寻找到目标概念的难度越大。对于有限的假设空间，可以用其中包含假设的数目来刻画假设空间的复杂度。然而，大多数学习问题考虑的假设空间是**无限的**（如实数域中的区间、$\mathbb{R}^d$ 空间中的线性超平面），因此无法直接使用假设的数目来刻画复杂度。

本章介绍**刻画无限假设空间复杂度的方法**，这是从理论层面理解机器学习算法学习能力的关键一步。复杂度度量不仅决定了学习算法的泛化性能，也为算法设计和模型选择提供了理论指导。

### 3.1 两种刻画思路的对比

| 特性 | VC维 (数据分布无关) | Rademacher复杂度 (数据分布相关) |
|------|---------------------|--------------------------------|
| **定义** | 能被打散的最大样本集大小 | 函数空间与随机噪声的相关性 |
| **数据依赖** | 与数据分布 $\mathcal{D}$ 无关 | 依赖于具体数据分布 |
| **计算难度** | 相对简单 | 较为复杂 |
| **紧度** | 较"松"，普适性强 | 较"紧"，针对性强 |
| **适用场景** | 理论分析、通用界限 | 具体问题、精确估计 |

### 3.2 核心定义

**VC维 (Vapnik-Chervonenkis Dimension)**:
$$VC(\mathcal{H}) = \max\{m: \Pi_{\mathcal{H}}(m) = 2^m\}$$

**增长函数 (Growth Function)**:
$$\Pi_{\mathcal{H}}(m) = \max_{\{\boldsymbol{x}_1, \ldots, \boldsymbol{x}_m\} \subset \mathcal{X}} |\{(h(\boldsymbol{x}_1), \ldots, h(\boldsymbol{x}_m)) \mid h \in \mathcal{H}\}|$$

**经验Rademacher复杂度 (Empirical Rademacher Complexity)**:
$$\hat{\mathfrak{R}}_Z(\mathcal{F}) = \mathbb{E}_{\sigma}\left[\sup_{f \in \mathcal{F}} \frac{1}{m}\sum_{i=1}^m \sigma_i f(z_i)\right]$$

**Rademacher复杂度 (Rademacher Complexity)**:
$$\mathfrak{R}_m(\mathcal{F}) = \mathbb{E}_{Z \subset \mathcal{Z}: |Z|=m}\left[\hat{\mathfrak{R}}_Z(\mathcal{F})\right]$$

## 4. 核心定理清单

| 定理 | 名称 | 核心内容 |
|------|------|----------|
| **引理3.1 (Sauer引理)** | Sauer-Shelah引理 | $\Pi_{\mathcal{H}}(m) \leq \sum_{i=0}^d \binom{m}{i}$ |
| **定理3.1** | 增长函数上界 | $\Pi_{\mathcal{H}}(m) \leq (em/d)^d$ |
| **定理3.2** | VC维与Natarajan维的关系 | $K=2$ 时两者相等 |
| **定理3.3** | 多分类增长函数上界 | $\Pi_{\mathcal{H}}(m) \leq m^d K^{2d}$ |
| **定理3.4 (Massart引理)** | Massart's Lemma | $\mathbb{E}_{\sigma}\left[\frac{1}{m}\sup_{\boldsymbol{x}\in A}\sum_{i=1}^m \sigma_i x_i\right] \leq \frac{r\sqrt{2\ln|A|}}{m}$ |
| **推论3.1** | Rademacher与增长函数关系 | $\mathfrak{R}_m(\mathcal{H}) \leq \sqrt{\frac{2\ln\Pi_{\mathcal{H}}(m)}{m}}$ |
| **定理3.5** | 齐次线性超平面VC维 | $VC(\mathcal{H}) = d$ |
| **定理3.6** | 非齐次线性超平面VC维 | $VC(\mathcal{H}) = d+1$ |
| **定理3.7** | 线性超平面Rademacher复杂度 | $\hat{\mathfrak{R}}_D(\mathcal{H}) \leq \sqrt{\frac{r^2\Lambda^2}{m}}$ |
| **定理3.8** | SVM的VC维上界 | $d \leq r^2\Lambda^2$ (与维数无关) |
| **定理3.9** | 神经网络VC维上界 | $VC(\mathcal{F}) = O(N\log_2 N)$ |

## 7. 一句话核心要义

> **复杂度度量是连接假设空间表达能力与学习算法泛化性能的桥梁：VC维提供了分布无关的普适刻画，而Rademacher复杂度则提供了分布相关的精确分析。**

## 8. 本章小结与反思

本章的核心贡献在于为**无限假设空间**提供了严格的复杂度刻画工具：

1. **VC维**通过"打散"概念将无限问题转化为有限分析，Sauer引理揭示了增长函数从指数增长到多项式增长的转折点。

2. **Rademacher复杂度**引入了数据分布的考量，通过与随机噪声的相关性来度量函数空间的丰富程度。

3. 两种方法各有优势：VC维计算简单、普适性强；Rademacher复杂度更紧、更贴近实际数据。

**反思问题**：
- 为什么VC维的定义要求"存在"一个被打散的样本集，而不是"所有"样本集都能被打散？
- Rademacher复杂度中的随机变量$\sigma_i$为什么取值为$\{-1, +1\}$？如果改用其他分布会有什么影响？
- 在实际应用中，如何权衡使用VC维还是Rademacher复杂度进行模型选择？
